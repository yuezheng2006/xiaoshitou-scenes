# 使用示例

## 生图入口（Codex）

安装 Skill 后，直接用中文说意图即可触发。

### 双 IP 入口（推荐 — 老杨 × 小石头）

```text
老杨：这篇内容想配图，请先推荐几张、什么类型，不要直接全生成。

<粘贴内容>
```

```text
老杨：把这套『四步内容复用方法』做成 4:5 知识卡，我主讲，小石头在步骤里搬卡和标风险。
```

```text
老杨和小石头：解释这个 RAG 工作流，出一张 16:9 白板图。
```

```text
yuezheng2006：PPT 演讲模式，主题《AI 时代内容怎么配图》，8 页。先出导演规划卡。
```

### 小石头单 IP 入口

推荐明确写：

- `小石头实物图：...`
- `小石头手绘图：...`

不必写配色、角色细节，也不必写 `Use $scene-skill-core`。

```bash
# 安装（一次性）
cp -R ./scene-skill-core "${CODEX_HOME:-$HOME/.codex}/skills/"
```

## 常用提示词

### 先规划，暂不生图

```text
老杨：先不要生图。请分析下面这篇中文内容，推荐双 IP 配图方案（模式、张数、老杨和小石头各自做什么）。

<粘贴内容>
```

```text
先不要生图。请分析下面这篇中文内容，分别给出小石头实物图和小石头手绘图的配图方案建议。

<粘贴内容>
```

### 生成一组实物图

```text
小石头实物图：把下面这篇内容生成 4 张 16:9 正文配图。

<粘贴内容>
```

### 生成一张手绘图

```text
小石头手绘图：把下面这个 Agent 工作流解释成一张 16:9 白板图。

<粘贴流程或说明>
```

### 单个观点一张实物图

```text
小石头实物图：为这个观点生成一张 16:9 正文配图：

产品功能很多，但用户真正记住的往往只有一个瞬间。
```

### 方法论一张手绘图

```text
小石头手绘图：把这个方法论画成一张结构清楚但不 PPT 的解释图：

先收集素材，再沉淀成可复用模块，最后组合成稳定输出。
```

### 知识卡（双 IP）

```text
老杨：把这套『四步内容复用方法』做成 3:4 收藏图。我主讲和批注，小石头在步骤里分工：搬卡、贴标签、标风险。

<粘贴方法内容>
```

### 知识卡（单 IP）

```text
小石头知识卡：把这套『四步内容复用方法』做成一张 3:4 收藏图，安排 2-3 个小石头在步骤里分工协作。

<粘贴方法内容>
```

### PPT 演讲模式

```text
PPT 演讲模式：主题《AI 时代，内容怎么配图才有效》，8 页直播分享。先出整套导演规划卡，确认后再逐页出图。

大纲：
1 封面 2 大判断：好配图服务读者 3 痛点：三个常见困境 4 场景：从长文留3个共鸣点
5 方法：配图三步 6 案例：一个处境一张图 7 时间线：试改留 8 收束：今晚只试一张
```

```text
小石头实物图：把这个项目复盘做成一张超横版长卷故事图。

主题：从一个本地想法，慢慢变成可规模化的产品。
节点：起点、原型、试点、反馈、规模化、新场景、持续迭代。
```

### 聚会娱乐 / 抢麦合唱


### 作者 × 小石头互动

```text

标签：先试 / 再改 / 留一版
```

```text
老杨：在白板旁解释双模式路由，小石头负责拉线连接实物图和手绘图。16:9 手绘图。
批注：实物图 / 手绘图 / 自动判断
```

```text
老杨：做一张 9:16 手机海报，核心观点是「先定角色，再拆内容，最后生成图解」。我主讲，小石头举执行卡。
```

### 改图

```text
这张小石头手绘图方向对，但文字太多。请减到 4 个短批注，其他保持不变。
```

```text
这张小石头实物图方向对，但主物件太重。请保持隐喻不变，把画面变轻一些。
```
