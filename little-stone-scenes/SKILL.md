# 小石头 IP 配图

## 触发方式

Skill 安装到 Codex 后，用户**直接用中文说意图即可**，例如「小石头实物图：为这篇内容出 4 张 16:9 配图」「小石头手绘图：解释这个流程」。Agent 应自动加载本 Skill，**不要**要求用户先写 `Use $little-stone-scenes`。

可选：`$little-stone-scenes` 仅用于用户想强制指定 Skill 时。

## 核心定位

把中文内容转译成两种清晰区分的 **小石头配图**：

```text
实物图：小石头 + 简笔物件 + 物理动作 + 短中文标签 + 留白叙事
手绘图：小石头 + 白板手绘结构 + 红橙蓝批注 + 核心概念动作
```

## 模式路由

显式风格词优先：

- 用户说 **实物图 / 实物场景 / 物件小现场**：使用 **实物图模式**。
- 用户说 **手绘图 / 手绘解释 / 白板图 / 逻辑图**：使用 **手绘图模式**。
- 用户说 **彩蛋 / 长卷 / 超横版 / 项目复盘 / 成长路径**：使用 **实物图长卷模式**。

未指定风格时，按内容类型判断：

- 处境、情绪、用户痛点、正文配图、项目故事、产品演化、K 歌服务现场、员工工牌、值班与服务工单：默认 **实物图模式**。
- 流程、结构、系统关系、方法论、认知拆解、工作流解释：默认 **手绘图模式**。
- 仍不确定时，先给 1-2 句说明并选择最贴合内容的模式，不要反复追问。

## 两种模式

- **实物图模式**：16:9 正文图，一个简笔物件小现场；也支持彩蛋长卷故事图。参考 `physical-style-dna.md`、`physical-master-anchors.md`、`physical-object-patterns.md`。
- **手绘图模式**：16:9 白板手绘解释图；用 1 个核心结构和少量红橙蓝批注解释流程、系统或方法。参考 `handdrawn-style-dna.md`、`handdrawn-composition-patterns.md`、`handdrawn-qa-checklist.md`。

## 小石头 IP（执行摘要）

- **形体**：圆角胶囊 `#f39200`；flat 2D 简笔；锚点 `little-stone-ref.png`。
- **性格**：略萌、认真、略笨、有点倔——靠 **动作** 表达。
- **眼睛**：白圆眼 + 可选 tiny 瞳孔；**不要** Logo 护目镜眼。
- **身上**：禁止 Logo / 品牌名 / 商标标记。
- **多图**：2 张及以上锁定同一小石头（见 1C）。
- **多人同图**：2-4 个小石头可以同场，但要像同 IP 家族里的不同个体，体型、姿态、朝向、动作分工必须明显不同，不要像复制粘贴。

完整规则：`references/common-little-stone-ip.md`。

## 共同底线（两种模式都必须保留）

模式只改变表达载体，不改变小石头自己的识别。

- 必须是同一个 **小石头**：`#f39200` 胶囊体、白圆眼、细线肢体、flat 2D 简笔。
- 必须靠 **物理动作** 讲故事：拉、挡、推、扛、解、贴、塞、接、修、守。
- 必须使用 **短中文标签 / 批注**，不要大段解释。
- 必须有 **留白**，不要做成信息堆叠、海报或 PPT。
- 必须保留「认真、略笨、有点倔」的气质，不要变成可爱吉祥物、商业 mascot、3D 卡通或正式企业图标。
- 禁止把小石头身上贴 Logo、品牌名或任何商标。

## 双模式来源

- **手绘图模式** 承袭 Ian Xiaohei Illustrations 1.0：白底手绘解释图，适合拆流程、结构和方法。
- **实物图模式** 承袭 Ian Xiaohei Scenes 2.0：简笔物件小现场，适合表达处境、情绪和故事。
- 两种模式共享小石头 IP，但生成时必须二选一，不要把实物场景和手绘解释图混成一张。

## Core Flow

### 1. 消化内容

先提炼当前内容里的“处境”，不要急着找物件：

- 这张图要让谁产生共鸣？
- 他们正在被什么东西拉扯、筛选、压住、拽回、催促、重组？
- 抽象概念能变成什么物理动作？
- 哪 2-4 个短词能一秒戳中痛点？
- 哪个真实物件天然承载这个处境？

优先选择高共鸣节点：会议、消息、返工、审查、筛选、加码、被替代、下班失败、项目卡住、内容资产沉淀、产品演化断点。

如果用户要求彩蛋模式，先把内容拆成 5-8 个连续节点：

- 左侧起点：身份、起点、最初处境。
- 中间节点：关键经历、项目转折、内容资产、产品演化。
- 右侧终点：现在在做什么、核心关注、结论或下一阶段。
- 每个节点都要有一个真实物件、小石头动作和 1-3 行短注释。

完成标记：已经明确读者处境、核心冲突、真实物件、小石头动作和短中文标签；彩蛋模式已经得到 5-8 个可视化节点。

### 1B. 事实锚定

涉及个人经历、品牌名、公司名、项目名、粉丝数、时间跨度、成绩数字时，只能使用用户输入、用户提供素材或用户明确确认的事实。

- 用户没有提供的事实，不要从母版继承，也不要补成看起来更完整的履历。
- 无法确认的内容，用概括性标签替代，例如“内容平台”“项目节点”“用户反馈”“产品实验”。
- 如果用户要求保留但信息不完整，在 shot list 或交付说明里标注“待确认”，不要把它画成确定事实。

完成标记：所有具体事实都能回到用户输入或素材；不能回溯的事实已删除、抽象化或标注待确认。

### 1C. Character Lock（2 张及以上强制）

一次任务要生成 2 张及以上配图时，必须先读取 `references/common-little-stone-ip.md`，写出全批共用的 Character Lock：

```text
Character Lock：
- 小石头：#f39200 + 胶囊轮廓 + 白圆眼（对照 little-stone-ref.png）
- flat 2D 简笔
- 锚点：little-stone-ref.png
```

生成后对照 `little-stone-ref.png` 及同批上一张：必须是 **同一小石头**。

完成标记：Character Lock 已声明；同批每张候选图已通过小石头一致性比对。

### 1C-2. 多人同图差异锁（多人场景强制）

当用户要求 **多人场景 / 多个 Agent / 团队协作 / 接力 / 评审会 / 结对 / 三方对齐** 时，可以画 2-4 个小石头。此时不是简单复制同一个小石头，而是同一 IP 下的不同个体。

生成前必须写出：

```text
Multi-person variation lock：
- 共享识别：#f39200 橙身、白圆眼、细线肢体、flat 2D、无 Logo
- 个体 A：{身高/胖瘦/朝向/姿态/动作职责}
- 个体 B：{身高/胖瘦/朝向/姿态/动作职责}
- 个体 C（如有）：{身高/胖瘦/朝向/姿态/动作职责}
```

差异要求：

- 至少 3 个维度明显不同：身高、胖瘦、倾斜角度、朝向、手脚姿态、站位前后、动作职责、所持道具。
- 每个小石头都要有明确分工：递卡、接卡、扶线、贴标签、看反馈、按住物件、拉住流程。
- 体型差异要比“同一素材平移”更明显，但仍保持胶囊 / 竖椭圆小石头识别。
- 禁止 clone stamp / copy-paste 感：同高同胖、同角度、同表情、同手脚姿势只是换位置，即失败。

完成标记：多人同图已通过差异锁；既像同一个 IP 家族，又不像复制粘贴。

### 1D. 作者出镜（可选）

只有用户明确提到 **老杨 / 作者 / yuezheng2006 / 我本人 / 我的数字形象 / 让我和小石头一起** 时，才加入作者数字形象。

触发后必须读取：

- `references/common-author-persona.md`
- `assets/persona/author-persona-ref.jpg`

作者出镜规则：

- 老杨是作者数字形象，不是默认角色；未触发时不出现。
- 老杨只要求 **形似轻 Q**：成熟 35+、约 183cm / 85kg、更高大结实、略宽肩，短黑直发、黑框眼镜、浅色上衣，面容略成熟、亲和克制；实物图里画成轻 Q 3D 数字玩偶 / 软黏土感角色，有圆润体积和柔和阴影，不要变成扁平纸片人；也不要画得过年轻、过瘦、幼态、显老疲惫、卷发/蓬松发型、夸张肌肉人、半写实真人、照片级真人或二次元/儿童卡通。
- 老杨必须和小石头互动：递工具、看反馈、画批注、扶住物件、和小石头一起处理现场。
- 小石头仍然承担核心动作，作者不能替代小石头。
- 不要做成作者头像展示、个人品牌海报、专访封面或证件照；作者不要超过主视觉 30%-35%。
- 不要发明作者的单位、联系方式、履历或背书。

完成标记：作者形象已基于参考图，且与小石头和核心场景有明确互动。

### 2. 先出 shot list

如果用户说“分析 / 思考 / 怎么配图”，先给 shot list。每张写清：

- 使用哪张母版
- 图的主题
- 读者共鸣点
- 物理动作
- 真实主物件
- 小石头在做什么
- 中文短标签
- 为什么这张值得画

标准模式默认 4-8 张。彩蛋模式默认先出 1 张长卷母版效果图。用户明确要“看效果 / 输出 / 生成”时，直接生图，不要停在方案，但仍必须先在内部完成 2B 母版锁定。

完成标记：每个候选画面都有母版、主题、共鸣点、物理动作、真实主物件、小石头动作、短标签和成图理由。

### 2B. 母版锁定（标准模式强制）

标准模式每张图在生成前必须先完成母版锁定。读取 `references/physical-master-anchors.md`，从 `01-06` 里选一张最匹配的质量锚点。母版不是可复制构图；不要把案例的手机、纸条、沙漏、电脑、放大镜等空间拓扑直接搬过来。

必填字段：

```text
母版类型（01–06）：
Character Lock（2 张及以上）：
抽取的不变量：
当前内容的变异点：
画面 3 秒读懂句：
当前内容适配：
要避免的失败信号：
```

选择原则：

- 下班失败、报警、返工回流、问题打结：优先 `03-problem-knot-alert`。
- 消息涌出、任务过载：优先 `02-info-overload`。
- 会议 / 同步 / 对齐把人拉回：优先 `01-alignment-pull-in`。
- 审查 / 校对 / 兜底 / 返工检查：优先 `04-review-rework`。
- AI 自动化、岗位重组、身份标签变化：优先 `05-automation-relabel`。
- 筛选 / 过滤 / 关键词 / 机会流失：优先 `06-filter-loss`。

如果当前内容不完全匹配任何母版，也要选择最接近的一张作为比例、留白和信息密度标尺，然后重写物件、动作和标签。

变异要求：

- 至少 3 项和母版不同：主物件类别、空间方向、小石头动作、道具组合、标签位置、视角或叙事重心。
- 如果主物件和母版相同，空间关系和小石头动作必须明显不同。
- 如果空间关系和母版相近，主物件、动作和配件必须明显不同。
- 不能把母版案例做成换字版。第一眼像旧案例换主题，即失败。

硬性预算：

- 只能有一个核心物理动作。
- 只能有一个真实主物件或紧凑主物件组。
- 最多 1-2 个小配件。
- 最多 4 个短标签，优先 3 个。
- 禁止把主题里的所有名词都画出来。出现“元素清单化”时，先删减再生成。
- 单个深色大物件不能支配画面；手机、电脑、黑色盒子等深色主物件要缩小或换成更轻的实物锚点。
- 整体场景要比母版更轻或同等轻，不能更满、更大、更重。

完成标记：每张标准图都有明确母版、不变量、至少 3 个变异点、3 秒读懂句、适配点和失败信号；没有元素清单化或母版复刻化。

### 2C. 手绘图结构锁定（手绘图模式强制）

手绘图不使用 `01-06` 实物母版。生成前必须读取：

- `references/handdrawn-style-dna.md`
- `references/handdrawn-composition-patterns.md`
- `references/handdrawn-qa-checklist.md`

先选择一个结构类型：

- Workflow 流程：输入 -> 处理 -> 输出、自动化链路、Agent 工作流。
- 系统局部：信息来源、数据库、检索、渲染器、局部模块关系。
- 前后对比：混乱/有序、手动/自动、分散/收拢。
- 角色状态：用户痛点、工具太多、卡住到跑起来。
- 概念隐喻：黑盒、怪机器、信息仓库、内容工厂。
- 方法分层：能力栈、方法论、系统层级。
- 地图路线：从想法到上线、用户路径、学习路线。
- 小漫画分镜：2-4 个小场景表达过程变化。

必填字段：

```text
模式：手绘图
结构类型：
核心意思：
画面结构：
小石头动作：
建议元素：
中文批注：
避免的失败信号：
```

硬性预算：

- 一张图只解释一个核心结构。
- 只用 3-5 个核心模块或元素，最多 5-8 个短批注。
- 小石头必须承担概念动作，例如拉线、守门、搬运、修补、称重、分拣、塞入。
- 不要做成正式流程图、PPT、课程课件、复杂架构图或 UI 截图。
- 不要画真实物件摄影棚场景；那属于实物图模式。

完成标记：每张手绘图都有结构类型、核心意思、小石头概念动作、短批注和失败信号；没有 PPT 化、标题化或复杂架构化。

### 3. 单张生成

每张图单独生成，不要把多张拼成一张。

默认用 16:9 横版。提示词必须包含对应模式：

- 实物图：母版锁定与 Character Lock。
- 手绘图：结构类型锁定与 Character Lock。
- `#FFFFFF` 纯白背景。
- 小石头：`#f39200` 胶囊体；表情靠物理动作，汗滴 0–1 个即可。
- 实物图：真实物件小现场，白色摄影棚表面，真实物件光影统一；小石头本体保持 flat 2D 简笔。
- 手绘图：黑色手绘线稿结构，少量红 / 橙 / 蓝批注。
- 中等主体覆盖面积，但视觉重量轻。
- 禁止截图、UI、Logo、PPT、商业插画、贴图边界和大段解释。

完成标记：每张图都有独立提示词，且包含模式、比例、背景、小石头动作、短标签/批注、点缀和负面约束。

### 3B. 彩蛋长卷生成

当用户说“彩蛋模式 / 长卷故事图 / 超横版 / 个人经历 / 项目复盘 / 产品演化 / 成长路径 / 像这张长图”时，使用彩蛋模式。

彩蛋模式锁定 **长卷文字骨架**。结构核心：

- 超横版长卷，比例接近 `2.6:1` 到 `3:1`。
- 高级近白背景，接近白色但保留极轻摄影棚空气感；不要死白导致物件漂浮。
- 一条细黑色手绘弯曲路线贯穿全图。
- 5-8 个真实物件节点，每个节点有小石头参与动作。
- 节点沿路线形成自然、不规律的上下起伏，不能紧凑平铺成一条水平流程，也不能做成规律正弦波。
- 节点之间要有明显留白且不等距，视觉重心左右平衡，不能让大物件集中或压住画面。
- 路线节奏要像手画出来：平缓段、突然抬高、浅弯、较深下探和安静收尾可以混合出现；避免机械的高低交替。
- 左侧是起点身份/开场，右侧是现在关注/结论。
- 节点文字贴近物件，像直接写在空白里的手写人生注释，不是信息图模块。
- 节点文案默认不要放在便利贴、纸条、卡片或标签框里；彩色胶带、小圆点和短下划线只做点缀。
- 节点不用序号、编号圆点或步骤标记；靠路线、物件和短标签自然串联。
- 彩色胶带、小圆点、短下划线只做节奏点缀。

彩蛋模式保留长卷骨架，按当前内容替换节点；不要复刻样例母版中的他人经历、第三方平台或品牌名，除非用户提供等价事实。

完成标记：长卷提示词包含超横版比例、高级近白背景、曲线路径、5-8 个真实物件节点、小石头逐段参与、左起右收、无序号节点和事实锚定后的节点文案。

### 4. 使用母版类型

母版是**质量标尺与动作类型索引**，不是可复制布局。`assets/examples/` 内 PNG 为可选视觉锚点；缺失时用 `physical-master-anchors.md` 文字骨架。

出图必须对齐这些优秀标准：

- 场景覆盖面积。
- 留白和空气感。
- 真实物件光影。
- 小石头和物件的比例关系。
- 短标签数量。
- 点缀色密度。
- 动作类型，例如左右拉扯、中心审查、物件涌出、标签重命名、筛选过滤。
- 叙事关系，例如“被拉进去”“挡不住”“被审查”“被重命名”“被筛掉”。

当当前文章和某张母版的动作类型高度匹配时，应当使用它作为质量锚点，抽取动作不变量，再为当前内容写出至少 3 个变异点。模板的价值是帮助判断“这种内容适合什么物理关系”，不是提供可描摹布局。

真正要避免的是机械套模板：不看当前内容，直接照抄旧案例的物件组合、摆放关系、小石头动作和标签结构。每次都要让当前文章重新决定隐喻，至少重写标签语境，并根据内容调整动作、关系、角度或物件细节。

彩蛋长卷锁定文字骨架：左起点、曲线路径、节点起伏、小石头逐段参与、右侧收束。

完成标记：已说明使用哪张母版的空间关系，并明确哪些物件、动作、标签来自当前内容。

### 5. QA 和迭代

生成后按 `references/physical-qa-checklist.md` 检查。重点看：

- 候选图是否真的对齐了所选母版，而不只是包含相似元素。
- 候选图是否过度复制了所选母版的空间拓扑、道具组合和小石头动作。
- 候选图是否 3 秒读懂；如果需要解释主题才懂，就是失败。
- 真实物件有没有截图感或贴图边界。
- 场景有没有过大、过小、过满或过散。
- 小石头：`#f39200` + 白圆眼；动作与场景同向。
- 同批 Character Lock（若多张）。
- 中文短标签是否准确、少、能读。
- 有没有把模板母版机械套用，而不是根据当前内容重新适配。
- 第一眼是否像“真实物件小现场”，而不是教程页、信息图或素材拼贴。
- 彩蛋模式是否形成一条连续的真实物件人生线：左侧起点、曲线路径、节点起伏、小石头逐段参与、右侧收束。

第一张生成图永远只是候选图，不是交付图。必须先查看候选图，并和所选母版做视觉比对。只要出现以下任何问题，就不交付，先重写提示词或重生成：

- 元素清单化：把主题名词都堆进去，缺少一个清楚的物理冲突。
- 母版复刻化：主体、方向、配件、小石头动作和标签位置像旧案例换字。
- 读不懂：不看说明时，画面冲突不清楚。
- 主物件过大或靠边，像商品图。
- 物件太多，像素材拼贴。
- 文字标签变成概念说明或标题。
- 小石头没有承担母版级的物理动作。
- 画面第一眼不像 `assets/examples/` 的正文配图质量。

不合格就重生成或局部编辑，不要把低质量图展示为最终交付。可以向用户说明“候选图未过 QA，正在重做”，但不要把它包装成效果展示。

完成标记：已通过 CRITICAL 项；未过 STANDARD 项时，交付状态标为 DONE_WITH_CONCERNS 并指出下一轮迭代方向。

## 输出口径

生成前：短而准，给 shot list 或生成计划。

生成后：展示图片、说明用途、给保存路径，并标出哪些最稳、哪些可选或需要再收一轮。

不要长篇解释理论。让图先说话。

## Completion States

| 状态 | 含义 |
| --- | --- |
| DONE | 已完成 shot list 或图片生成，并按 QA 检查通过 |
| DONE_WITH_CONCERNS | 已交付，但存在文字稳定性、物件真实感、背景纯度/近白空气感等待优化项 |
| BLOCKED | 缺少主题/正文/节点，或用户要求的事实、人物、品牌信息不可确认 |

## Quality Gates

### Gate 1：CRITICAL（不过 = 不交付为最终图）

- 标准模式必须有母版类型锁定；彩蛋模式必须锁定彩蛋长卷骨架（无编号母版，见 `physical-style-dna.md`「彩蛋长卷故事图」）。
- 标准模式必须有至少 3 个当前内容变异点。
- 小石头必须出现，承担核心物理动作；形体对照 `little-stone-ref.png`。
- 一次任务生成 2 张及以上时，全批须同一 Character Lock。
- 真实物件必须像同一白色摄影棚里的真实小现场，不能是截图拼贴。
- 标准模式必须是 16:9 正文图；彩蛋模式必须是超横版长卷图。
- 标准模式背景必须接近 `#FFFFFF`；彩蛋模式背景必须是干净高级近白。
- 中文标签必须短、少、可读，不能是假中文或大段解释。
- 不得出现元素清单化：把主题里的多个名词、多个物件、多个标签都堆在一张图里。
- 不得出现母版复刻化：把案例里的物件、方位、配件和小石头姿态原样换文字。
- 画面必须 3 秒读懂；如果必须靠交付说明解释隐喻，不能交付为最终图。
- 不得加入用户未提供或未授权的人名、联系方式、公司名、Logo、隐私信息。

### Gate 2：STANDARD（不过 = 标注后可继续迭代）

- 画面比例、留白、场景覆盖面积对齐母版。
- 小石头形体稳定，不像表情包；动作与标签一致。
- 真实物件不压画面，节点之间有空气。
- 彩色点缀只做节奏，不抢主体。
- 输出能说明“哪张最稳、哪张需要再收一轮”。

## Good vs Bad

**好的：**

> 一个真实主物件 + 一个小石头物理动作 + 2-4 个短标签。

**差的：**

> 把观点做成 PPT 流程图，堆很多箭头、模块框、大标题和解释文字，小石头只是站在旁边。

**差在哪：**前者靠物理动作和真实物件讲故事；后者靠信息堆叠解释概念。

## Suppressions

- 用户明确提供的个人经历、品牌名、项目名，不算“未授权个人信息”，但不要额外发明。
- 母版里的具体物件可以在内容匹配时复用，不算机械复刻；机械复刻指不看当前内容直接照搬旧图。
- 生图模型偶发的轻微文字不稳不等于 Skill 失败，但必须在交付时标注并建议迭代。
