# QA Checklist

## 必过项

### 母版与构图

- 标准模式已锁定 01–06 母版类型；PNG 有则比对，无则用 `physical-master-anchors.md` 骨架。
- 16:9 纯白 `#FFFFFF`；彩蛋长卷近白；留白足够。
- 场景覆盖约宽 60–72%、高 38–50%；3 秒读懂一个物理冲突。
- 无元素清单化、无母版换皮复刻。

### 真实物件现场

- 真实物件清晰、完整、自然，像同一张白色摄影棚照片里的小现场。
- 物件之间光影、透视和轻接触阴影一致；无 UI 截图、App 界面或拼贴边界。
- 主物件必须有真实物件质感和台面关系，不是平面图标或白板线稿。
- 小石头必须和主物件发生接触：挡、推、拉、扛、解、改、接、检查。
- 画面第一眼应像「简笔实物小摆件现场」，不是白板手绘解释图。

### 小石头 IP

- 形体对照 `little-stone-ref.png`，皮肤 `#f39200`。
- 白圆眼 + 可选 tiny 瞳孔；**不要** Logo 护目镜眼、不要 anime 眼。
- 表情靠 **动作**，汗滴 0–1 个即可。
- **身上** 无 Logo / 品牌名 / 商标标记。
- 2 张及以上：全批同一小石头 + flat 2D。
- 多人同图：同 IP 但不是复制粘贴；每个小石头的体型、倾斜角度、朝向、手脚姿态、站位层次和动作职责至少有 3 项明显不同。

### 合规

- 不出现未请求的公司名、Logo、商标、联系方式或个人隐私。
- 如果用户明确要求真实品牌元素，交付时标注由用户自行确认授权。

## 失败信号

- 不像小石头 ref（Logo 眼、星火体、新 mascot 物种）。
- 过度细节、商业海报、海外 editorial 风。
- 小石头只站看、不承担动作。
- 同批形体/画法不一致。
- 多人场景里的小石头同高同胖、同角度、同手脚姿势，只是平移复制。
- 身上误印 Logo、品牌名或商标；旁边有箭头指向小石头解释品牌身份。
- 堆很多五官细节，与物理动作脱节。
- 主物件只是平面线稿、图标或纸片符号，没有真实物件质感和台面关系。
- 画面靠箭头和文字解释，像手绘图模式，而不是物理现场。

## 交付判断

1. 第一眼：简、干净。
2. 第二眼：看懂物件与小石头的物理冲突。
3. 标签：共鸣。

不合格 = 重生成，不要交付。
