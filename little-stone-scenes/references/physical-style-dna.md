# 风格 DNA

## 一句话

白色空间里的真实物件小现场：小石头在里面认真做一件荒诞但成立的事，用少量中文手写词把读者的处境点破。

## 核心公式

```text
一个真实主物件 / 一个小石头核心动作 / 2-4 个短标签 / 少量彩色点缀 / 足够留白
```

## 画面比例

2.0 有两种规格：标准 16:9 正文配图和彩蛋长卷故事图。

### 标准 16:9 正文配图

标准 2.0 正文配图不是极简单物件，也不是满版信息图。

- 16:9 横版。
- 纯白背景，白色摄影棚表面。背景必须是接近 `#FFFFFF` 的干净纯白，不要暖白、灰白、米白、淡灰渐变或四周暗角。
- 场景外轮廓通常覆盖画面宽度的 60%-72%。
- 场景高度通常覆盖画面高度的 38%-50%。
- 视觉重量中等偏轻：不是把整体缩小，而是让物件之间有空气。
- 画面上方、四周和物件之间要有明显留白。留白区域必须保持纯白，不要被环境光、阴影或渐变染灰。
- 避免 close-up。真实物件不能大到压迫画面。
- 避免 miniature。场景不能小到信息不足。

判断句：

```text
占比够，但不挤；物件铺开，但不重；一眼有现场感，但不是教程页。
```

### 彩蛋长卷故事图

彩蛋模式是“小石头长卷故事图”：把个人经历、项目复盘、产品演化、成长路径和内容资产演进，画成一条横向展开的真实物件人生线。

彩蛋长卷使用 **长卷文字骨架**（公开版不包含长卷 PNG 母版；按本节与 `physical-master-anchors.md` 骨架执行）：

必须锁住的结构：

- 超横版长卷，比例接近 `2.6:1` 到 `3:1`。
- 左侧是身份 / 起点 / 开场文案。
- 一条细黑色手绘路线从左到右弯曲穿过全图。
- 5-8 个真实物件节点沿路线分布，有自然、不规律的高低起伏；不能排成一条平直横线，也不能做成规律的正弦波。
- 节点之间要松开且不等距，不能紧凑堆满，也不能均匀排布；每个物件、文字和小石头之间都要有呼吸感。
- 路线节奏要像手画出来的真实路径：可以有一段近乎平直的缓行、一段突然抬高、一段浅弯、一段更深的下探和一个安静收尾；避免“上、下、上、下”的机械交替。
- 画面视觉重心要左右平衡：左侧有起点，中段有一两个主要峰谷，右侧有收束；不要把大物件集中在同一侧。
- 每个节点都有小石头参与动作，不是站在旁边装饰。
- 节点文字贴近物件，像直接写在空白里的手写人生注释或项目注释。
- 节点文案默认不要放在便利贴、纸条、卡片或标签框里；可以偶尔用彩色胶带、小圆点、短下划线做节奏点缀，但文字本身应自由落在画面空白中。
- 节点不要加序号、编号圆点、步骤号或时间轴标记；顺序由路线自然带出来。
- 右侧是当前关注、结论或下一阶段。

彩蛋模式使用**高级近白背景**，不是强制纯白。长卷留白很大，死白会让物件和文字像漂浮在网页上。背景应该接近白色，但保留极轻的摄影棚空气感：

- 主色接近 `#FAFAF8` / `#FBFBFA`。
- 不能变成明显灰白、暖黄、米色、纸纹或暗角。
- 中央和文字附近必须干净。
- 可以有非常轻的柔光层次和物件接触阴影。
- 不能有海报渐变、摄影棚灰阶、四周暗边或脏背景。

判断句：

```text
像一条轻轻展开的白色展台人生线：有连续路径、有真实节点、有空气感、有阶段记忆。
```

## 真实物件规则

必须像同一张白色摄影棚照片里发生的事：

- 主物件真实、清晰、完整，不能有明显裁切边界。
- 所有真实物件共享一致光影、透视和软阴影。
- 阴影只能是物件脚下很轻的接触阴影，不能形成整张图的灰色背景、暗角或大面积渐变。
- 不要出现矩形截图贴片、UI 截屏、App 界面、未请求的 Logo、未提供或未授权的个人信息和公司名。
- 真实物件不应显得像图库素材随便拼上去。
- 标准 16:9 每张只保留 1 个主物件或 1 个紧凑主物件组，可加 1-2 个小配件，例如回形针、长尾夹、便签、胶带、杯子、沙漏、印章。
- 彩蛋长卷每个节点只保留 1 个主物件或 1 个紧凑主物件组，全图靠 5-8 个节点形成连续故事。
- 小石头本身仍是 **flat 2D 简笔角色**，但必须和真实物件发生可见的物理接触。
- **2D Flat Lock**：小石头 = 平涂 `#f39200` 贴纸 / 纸片人；身体无渐变、无高光、无体积阴影；四肢是细黑线，不是 3D 圆管；眼睛是平面白圆，不是反光球体。真实物件可以有摄影棚光影，**小石头不能继承 3D 渲染语言**。

失败信号：

- 只有黑色线稿和手写箭头，看起来像白板手绘图。
- 主物件不是真实物件，只是平面图标或纸片符号。
- 小石头和物件没有接触关系，只是站在旁边说明。
- 箭头/流程线成为主体，而不是物理动作成为主体。

## 信息密度

高质量 2.0 图应该“有信息，但不解释完”：

- 标准 16:9：1 个核心物理动作、1 个小石头动作主体、2-4 个短中文标签（可写在便签/纸条/气泡上）、2-4 个小纸条或标签纸视觉元素、4-6 个彩色点缀。
- 彩蛋长卷：5-8 个节点，每个节点 1 个小石头动作和 1-3 行短注释；全图点缀跟随路线节奏，不平均撒满。
- 不画完整流程，不画全链路，不画所有因果；彩蛋长卷画的是阶段记忆线，不是流程说明书。

## 中文标签

中文标签是情绪按钮，不是说明书。

- 标准 16:9 每张 2-4 个，优先 3 个。
- 彩蛋长卷每个节点 1-3 行短注释，整体文字保持轻，不要把每个节点写成段落。
- 彩蛋长卷节点标签不编号，避免把人生线变成流程步骤图。
- 每个 2-6 个字，尽量短。
- 写高频真实话术，例如“对齐一下”“马上回”“谁改的”“小问题”“AI 先做”“人呢”。
- 标签要像便利贴、手写批注、纸条上的字。
- 不要大段解释、长标题、结构名称。
- 不要左上角写“流程图 / 工作流 / 系统架构 / 常见坑”等类型标题。

## 颜色

颜色只做点缀，不做主视觉。

- 品牌橙 `#f39200`：**小石头皮肤**。
- 黑色：手写线、核心文字、牵引线（不再作为 IP 主色）。
- 蓝色：胶带、补充状态、系统感下划线。
- 粉色：小胶带、柔和提示。
- 黄色：便签、标签、圆点、轻提醒。
- 绿色：小圆点、通过/状态感。
- 红色 / 橙红：痛点、报警、重点下划线、情绪钉子。

一张图通常 4-6 个彩色小点缀就够。彩色面积太大，会显得廉价。

## 高级感来自哪里

- 真实物件光影统一。
- 留白足够，物件之间有空气。
- 标签少而准。
- 小石头动作准确；形体对照 ref 稳定；**必须保持 flat 2D 平涂，禁止 3D 公仔化**；表情靠动作与 0–1 汗滴即可，不堆五官。
- 物理隐喻清楚，但不把道理讲满。
- **不像**海外商业插画或 3D 产品渲染；**小石头尤其不能像 3D mascot / 软陶公仔**。

## 绝对不要

- 不要 PPT 信息图、流程图、课程课件。
- 不要精致商业插画、海外 editorial 风、3D 渲染、写实产品摄影、科技感 UI。
- 不要复杂背景、办公室场景、城市夜景、渐变、纸纹、噪点、复杂光影。
- 标准 16:9 不要灰白背景、暖白背景、背景渐变、暗角、摄影棚灰阶或大面积环境阴影。
- 彩蛋长卷不要死白、脏灰、暖黄、暗角或海报渐变；保留高级近白空气感。
- 不要长篇文字。
- 不要多个场景堆在同一张图。
- 不要让小石头像吉祥物或表情包；**不要**为表情把胶囊体拉形或换成 Logo 导出轮廓。
- **不要**把小石头画成 3D 角色：禁止 clay/vinyl/Pixar/盲盒/充气玩具/渐变高光/体积阴影/圆管 3D 四肢。
- 不要堆复杂五官；表情须与物理动作 / 标签情绪一致。
- 不得把完整麦克风或头套麦替代小石头本体。
- `assets/examples/` 母版 PNG 为可选视觉锚点；**缺失时**以 `physical-master-anchors.md` 抽象骨架 + 本文比例规则 + `assets/brand/` 为准。有 PNG 时须对齐比例、留白、动作清晰度；不要机械照抄旧案例物件组合。
- 彩蛋长卷锁定 **长卷骨架（无编号母版）**（见上文「彩蛋长卷故事图」）；不锁定样例中的第三方物件与文字。
