# 真实物件模式与原创规则

## 常用结构

### 0. 彩蛋长卷故事线

适合：个人经历、项目复盘、产品演化、成长路径、内容资产演进、一个人从旧身份走到新方向。

彩蛋长卷使用文字骨架（公开版不包含长卷 PNG 母版；用 `physical-style-dna.md` 长卷结构）：

画法：

- 超横版长卷，接近 `2.6:1` 到 `3:1`。
- 左侧放身份 / 起点 / 开场，不做大标题海报。
- 一条细黑色手绘路线从左到右穿过全图。
- 5-8 个真实物件节点沿路线排布，节点有高低起伏。
- 每个节点：小石头 + 物理动作。
- 节点旁边写 1-3 行短注释，像手写人生注释。
- 右侧收束到“现在在做什么 / 当前关注 / 下一阶段”。
- 背景是高级近白空气感，像轻轻展开的白色展台。

避免复刻：不要照搬样例长卷里的他人平台名、品牌名、具体物件组合。只学习长卷骨架：左起、曲线、节点、物件、小石头、右收。

### 1. 拉扯现场

适合：会议、需求、下班失败、项目推进。

画法：

- 真实主物件在一侧。
- 小石头被线、纸条、挂绳、胶带拉向主物件。
- 标签散在拉扯来源附近。

避免复刻：不要总画电脑和三张会议卡片。可以换成门把手、文件夹、快递单、白板擦、工牌绳、票据夹。

### 2. 涌出现象

适合：消息、任务、bug、素材、想法。

画法：

- 真实物件像一个出口。
- 纸条、小卡片、标签从物件里冒出来。
- 小石头挡、接、推回、整理或躲避。

避免复刻：不要总画手机。可以用抽屉、信封、打印机、杯子、文件盒、胶带座。

### 3. 审查现场

适合：Review、评审、修改、验收、挑错。

画法：

- 放大镜、印章、红笔、夹板、清单等真实物件。
- 小石头在物件下被放大、被盖章、被要求修改。
- 短标签像评审意见。

避免复刻：不要总画放大镜 + 印章。可以用夹子、尺子、红笔、标签机、打孔器、透明文件袋。

### 4. 重命名 / 重组现场

适合：AI 替代、岗位变化、身份变化、产品定位变化。

画法：

- 工牌、标签、贴纸、胶带、文件名牌。
- 小石头拉住、撕开、按住、藏起或重新贴上标签。

避免复刻：不要总画工牌 + 修正带。可以用文件夹标签、抽屉标签、快递面单、价格牌、行李牌。

### 5. 筛选现场

适合：简历、关键词、内容筛选、用户分层、机会过滤。

画法：

- 筛子、漏斗、夹子、秤、滤纸、抽屉。
- 小石头把卡片推入、从筛孔接住、被筛掉或守住入口。

避免复刻：不要总画金属筛子。可以用茶漏、漏斗、夹板、滤网、邮筒口、门缝、票据机。

### 6. 加码现场

适合：OKR、指标、压力、债务、任务堆叠。

画法：

- 秤、纸堆、砝码、夹子、弹簧、文件盒。
- 小石头扛住、被压住、扶住、称重。

避免复刻：不要总画厨房秤。可以用弹簧夹、杯垫、文件托盘、订书机、纸镇。

## 原创隐喻三步

1. 把观点写成冲突句：

```text
读者被 X 以 Y 的方式推向 Z。
```

2. 把 X/Y/Z 换成真实物件和动作：

```text
X = 物件 / 纸条 / 标签 / 线 / 筛子
Y = 拉、筛、压、贴、拽、打结、盖章
Z = 被吞进去、被拽回、被重命名、被筛掉
```

3. 让小石头承担动作，而不是旁观：

```text
小石头正在拉 / 挡 / 推 / 扛 / 解 / 改 / 撕 / 接。
```

## 模板母版使用规则

样例图是高质量模板母版和出图质量标尺。它们不是死板模板，也不是物件排除规则。

出图必须对齐模板的三层优秀标准：

- 空间比例：主体多大、留白在哪里、物件之间怎么留空气。
- 动作类型：左右拉扯、物件涌出、中心审查、标签重命名、筛选过滤、压力加码。
- 叙事关系：谁拉谁、谁挡谁、谁被筛、谁被贴标签、谁被拽回。

彩蛋模式沿用长卷文字骨架。做彩蛋长卷时必须保留左起、曲线、节点、物件、小石头、右收，但替换成当前内容自己的节点、物件、动作和文字。

当当前内容和母版动作类型高度匹配时，应当优先使用对应母版作为质量锚点；相似主物件可以出现，但必须同时改变空间关系、小石头动作、配件组合、标签位置或视角，避免变成旧案例换字。比如内容就是“手机消息”，手机当然可以出现；内容就是“Code Review”，放大镜、印章、红笔也可以出现。

不能做的是机械复刻：直接照搬旧图的物件组合、摆放、动作和标签。

使用模板时，至少做一次内容适配：

- 标签必须来自当前内容。
- 小石头动作必须服务当前内容的核心冲突。
- 物件关系要能解释当前内容，而不是为了像样例。
- 如果主物件相同，就优先调整动作、角度、纸条关系或标签语境。
- 如果动作类型相同，就优先调整主物件、空间关系、配件、标签位置或小石头姿态。

例：

- “手机消息”可以继续用手机，也可以使用“纸条涌出”的动作类型；但标签、动作、纸条关系、配件和布局要根据当前内容重写。
- “Code Review”可以继续用放大镜；但不一定要放大镜 + 印章，也可以变成红笔、夹板、标签机或透明文件袋。
- “AI 简历筛选”可以继续用筛子；但可以换成漏斗、门缝、票据机，也可以保留筛子但改变小石头动作。

| --- | --- | --- | --- | --- |
| 点歌排队 | 请求涌入、排队、插队、催促 | 点歌票据、号码牌、麦克风、便签 | 挡住 / 整理 / 递下一张 | 等一下 / 下一首 / 马上到 |

使用边界：

- K 歌服务图是**服务现场隐喻**，不是娱乐海报。
- 不画包厢实拍、舞台灯效、用户肖像、App 点歌界面或真实门店环境。
- Logo / 员工工牌只在用户明确要求雷石、员工、工牌、值班、服务身份时启用；详见 `common-logo-safety.md`。
- 小石头必须和服务物件发生物理接触：递票、挡单、插麦、贴房态、拉工牌、接工单。

## 物件选择原则

好物件要满足三条：

- 读者熟悉。
- 可以和小石头发生物理互动。
- 不需要 UI 截图也能表达主题。

可选轻娱乐锚点（不强制）：麦克风、便携音箱、耳机、遥控器、票据、歌单卡、房态牌、服务工单。

坏物件：

- 只有装饰价值。
- 太复杂，容易像素材拼贴。
- 太大，一出现就压住画面。
- 太科技，变成 AI 海报或产品广告。
