# Render Card · 生成卡

## 用途

生成卡是最终交给图像生成工具或人工执行者的明确指令。它只消费已经确认的任务卡和方案卡，不重新解释用户意图。

## 字段

```text
任务名称：
本张编号：
模式与比例：
背景与画面语言：
主角色 Profile：
主角色动作：
Persona 角色与动作：
主物件 / 主结构：
画面关系：
短中文标签：
参考资产及传图顺序：
硬性限制：
软性偏好：
负向约束：
最终 Prompt：
返修 Prompt 预案：
```

## 规则

1. 参考资产必须使用实际存在的路径。
2. 硬性限制必须来自 Profile、模式规则或用户明确要求。
3. Prompt 是 Render Card 的编译结果，不是新的事实来源。
4. 每张图单独生成，除非当前模式明确要求按页批量生成。
