# Task Card · 任务卡

## 用途

任务卡是一次配图任务的入口交接单。它只记录用户明确提出的需求、可追溯事实和当前需要确认的缺口，不提前决定构图细节。

## 何时使用

- 单张简单图：可以在内部快速填写，不必完整展示给用户。
- 多图、长文、Persona、品牌或事实较多的任务：生成前必须保留。

## 字段

```text
任务名称：
用户原话：
内容来源：正文 / 主题 / 节点列表 / 大纲 / 截图 / 其他
用户目标：
明确事实：
我的推断：
待确认信息：
Profile：default-little-stone / none / 其他
Persona：未触发 / 老杨双 IP / 小剧场 / 其他
用户明确要求生成：是 / 否
```

## 规则

1. 用户原话和 Agent 推断必须分开。
2. 未提供的个人经历、品牌事实、数字和履历不能写成确定事实。
3. Task Card 不负责选择母版、构图或 Prompt。
4. 如果缺少的信息不阻塞任务，使用概括性标签，不要反复追问。
