# 真实物件模式与原创规则

## 常用结构

### 0. 彩蛋长卷故事线

适合：个人经历、项目复盘、产品演化、成长路径、内容资产演进、一个人从旧身份走到新方向。

彩蛋长卷使用文字骨架（公开版不包含长卷 PNG 母版；用 `physical-style-dna.md` 长卷结构）：

画法：

- 超横版长卷，接近 `2.6:1` 到 `3:1`。
- 左侧放身份 / 起点 / 开场，不做大标题海报。
- 一条细黑色手绘路线从左到右穿过全图。
- 5-8 个真实物件节点沿路线排布，节点有高低起伏。
- 每个节点：主角色 + 物理动作。
- 节点旁边写 1-3 行短注释，像手写人生注释。
- 右侧收束到“现在在做什么 / 当前关注 / 下一阶段”。
- 背景是高级近白空气感，像轻轻展开的白色展台。

避免复刻：不要照搬样例长卷里的他人平台名、品牌名、具体物件组合。只学习长卷骨架：左起、曲线、节点、物件、主角色、右收。

### 1. 拉扯现场

适合：会议、需求、下班失败、项目推进。

画法：

- 真实主物件在一侧。
- 主角色被线、卡片、挂绳、轻薄标签拉向主物件。
- 标签散在拉扯来源附近。

避免复刻：不要总画电脑和三张会议卡片。可以换成轻薄项目卡、权限卡、会议纪要卡、磁吸白板条、协作看板卡、工牌挂绳、任务标签夹。

### 2. 涌出现象

适合：消息、任务、bug、素材、想法。

画法：

- 真实物件像一个出口。
- 小卡片、任务条、消息标签从物件里冒出来。
- 主角色挡、接、推回、整理或躲避。

避免复刻：不要总画手机。可以用消息收纳盒、任务队列卡、透明资料托、智能提醒牌、桌面小屏边缘、轻薄标签卷。

### 3. 审查现场

适合：Review、评审、修改、验收、挑错。

画法：

- 审阅卡、批注笔、轻薄清单板、透明标注片、标签机、验收贴纸等真实物件。
- 主角色在物件旁被标注、被圈改、被要求修改。
- 短标签像评审意见。

避免复刻：优先使用批注笔、轻薄清单板、透明标注片、标签机、验收贴纸、圆角评审卡，不回到传统审查道具组合。

### 4. 重命名 / 重组现场

适合：AI 替代、岗位变化、身份变化、产品定位变化。

画法：

- 工牌、标签、贴纸、胶带、文件名牌。
- 主角色拉住、撕开、按住、藏起或重新贴上标签。

避免复刻：优先使用文件标签条、权限卡、状态贴、资产编号牌、桌牌、圆角身份卡，不回到传统办公修补道具组合。

### 5. 筛选现场

适合：简历、关键词、内容筛选、用户分层、机会过滤。

画法：

- 智能筛选板、标签队列、权限闸口、评分卡、分层托盘、自动化流转槽。
- 主角色把卡片推入队列、从出口接住、被分层卡住或守住入口。

避免复刻：优先使用智能筛选板、标签队列、权限闸口、分层托盘、自动化流转槽、圆角评分卡，不回到传统过滤道具。

### 6. 加码现场

适合：OKR、指标、压力、债务、任务堆叠。

画法：

- 任务堆叠卡、指标看板、进度条压片、优先级卡、自动化队列、透明资料托。
- 主角色扛住、被压住、扶住、对齐优先级或守住队列入口。

避免复刻：优先使用指标看板、进度条压片、优先级卡、任务堆叠卡、自动化队列、透明资料托，不回到传统称重道具。

## 原创隐喻三步

1. 把观点写成冲突句：

```text
读者被 X 以 Y 的方式推向 Z。
```

2. 把 X/Y/Z 换成真实物件和动作：

```text
X = 物件 / 卡片 / 标签 / 线 / 队列 / 看板
Y = 拉、分层、压、贴、拽、打结、标注、圈改
Z = 被吞进去、被拽回、被重命名、被分层、被拦截
```

3. 让主角色承担动作，而不是旁观：

```text
主角色正在拉 / 挡 / 推 / 扛 / 解 / 改 / 撕 / 接。
```

## 模板母版使用规则

样例图是高质量模板母版和出图质量标尺。它们不是死板模板，也不是物件排除规则。

出图必须对齐模板的三层优秀标准：

- 空间比例：主体多大、留白在哪里、物件之间怎么留空气。
- 动作类型：左右拉扯、物件涌出、中心审查、标签重命名、筛选过滤、压力加码。
- 叙事关系：谁拉谁、谁挡谁、谁被分层、谁被贴标签、谁被拽回。

彩蛋模式沿用长卷文字骨架。做彩蛋长卷时必须保留左起、曲线、节点、物件、主角色、右收，但替换成当前内容自己的节点、物件、动作和文字。

当当前内容和母版动作类型高度匹配时，应当优先使用对应母版作为质量锚点；相似主物件可以出现，但必须同时改变空间关系、主角色动作、配件组合、标签位置或视角，避免变成旧案例换字。比如内容就是“手机消息”，手机当然可以出现；内容就是“Code Review”，可以用批注笔、评审卡、透明标注片或验收贴纸。

不能做的是机械复刻：直接照搬旧图的物件组合、摆放、动作和标签。

使用模板时，至少做一次内容适配：

- 标签必须来自当前内容。
- 主角色动作必须服务当前内容的核心冲突。
- 物件关系要能解释当前内容，而不是为了像样例。
- 如果主物件相同，就优先调整动作、角度、卡片关系或标签语境。
- 如果动作类型相同，就优先调整主物件、空间关系、配件、标签位置或主角色姿态。

例：

- “手机消息”可以继续用手机，也可以使用“消息卡涌出”的动作类型；但标签、动作、卡片关系、配件和布局要根据当前内容重写。
- “Code Review”可以用批注笔、评审卡、透明标注片或验收贴纸，保持轻量现代办公质感。
- “AI 简历筛选”可以变成智能筛选板、标签队列、权限闸口或自动化流转槽，保持数字化筛选语境。

使用边界：

- K 歌服务图是**服务现场隐喻**，不是娱乐海报。
- 不画包厢实拍、舞台灯效、用户肖像、App 点歌界面或真实门店环境。

- 禁止把麦克风画成舞台化、直播化或专业录音棚化的道具，除非用户明确要求。

## 物件选择原则

好物件要满足三条：

- 读者熟悉。
- 可以和主角色发生物理互动。
- 不需要 UI 截图也能表达主题。
- 默认现代化、轻量化、干净：优先近几年常见的无线设备、圆角卡片、哑光小物、磁吸底座、便携收纳盒、轻薄标签、简洁夹扣。
- 避免复古化、笨重化和舞台化：除非用户明确要求，不选年代感强、脏乱、厚重或商业舞台感过强的道具。

可选轻娱乐锚点（不强制）：现代无线麦、短柄便携麦、磁吸充电座、家庭 K 歌音箱、车载 K 歌音箱、便携音箱、耳机、遥控器、点歌器、票据、歌单卡、包房号牌、房态牌、服务票、路线卡、大中控屏边缘、抽象歌词条、无线充电板、中控岛、杯架、中央扶手、座椅缝、车门储物格、氛围灯条、无 Logo 方向盘局部、沙发靠垫、歌词牌、曲库盒、标签机、设备卡、同步贴纸、连接线、铃鼓、彩纸点。
