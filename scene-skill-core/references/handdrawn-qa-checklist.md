# 手绘图 QA Checklist

## 形象检查（先于模式 QA，CRITICAL）

**每一张候选图**交付前必做；**全部 PASS 后**输出 `common-character-lock.md` 的 **Confirm Gate** 结论，只有 `CONFIRMED` 才可交付。

1. **小石头 · 四肢 L1–L4**：计数 2 臂 2 腿；臂锚点体侧上 1/3、腿锚点底缘；细黑线连续、无 3D 圆管；无分叉指/绳线并手；**无嘴**。
2. **小石头 · 眼面 E1–E2**：对照 `primary-character-reference.png`；白圆眼+可选 tiny 瞳孔；位置/大小/间距批内一致；胶囊宽高比一致。
3. **老杨 P1–P7（若出现）**：对照 `author-persona-spec.png` + `author-persona-handdrawn.png`（线稿风）；六项识别 + 表情稳定域；正面或 3/4 可见脸。
4. **跨图稳定（同批 2 张+）**：小石头眼法/线宽/无嘴一致；老杨眼镜/发型/唇厚/穿搭一致。
5. 任一 FAIL → REJECT，不交付。

## 必过项

- 是 16:9 横版。
- 背景是干净纯白。
- 有主角色，且主角色承担核心概念动作。
- **主角色必须是 flat 2D 平涂简笔**：纯色 `the profile primary color` 主色身体、细黑线四肢、平面白圆眼；禁止 3D 渐变、高光、软陶/公仔/Pixar 质感。主色身体必须是一块均匀纯色，不允许轻微明暗渐变或体积阴影。
- **肢体数量正确**：每个主角色只能有两只手臂、两条腿；线条、箭头、绳子不能粘成额外手臂。
- 画面是白板手绘解释图，不是实物小现场。
- 没有复刻旧案例构图，而是为当前内容生成了新隐喻。
- 一张图只讲一个核心结构。
- **装饰性测试**：去掉主角色（及出镜的persona），画面结构是否还能讲清楚？如果完全讲得清楚，说明角色是装饰，判不合格（详见 `common-qa-repair.md`）。
- 主体占画面约 40%-60%，至少 35% 留白。
- 中文标注少、短、能读，最多 5-8 处。
- 橙色只用于主路径、箭头或关键流向。
- 红色只用于重点、问题、提醒或结果。
- 蓝色只用于补充说明、反馈或系统状态。
- 不出现真实 UI 截图、App Logo、代码编辑器截图或仪表盘。
- 不把参考资产文件名、内部路径或 prompt 片段画进图里；如 `author-persona-spec.png`、`author-persona-handdrawn.png` 只能作为生成参考，不能成为画面文字。
- 多人 / 多个主角色手绘图中，所有主角色共享同一 IP 与**相同胶囊宽高比**；只允许整体缩放、倾斜、朝向、手脚姿态、结构位置和概念动作不同；**禁止**瘦高/矮胖/横扁形体混用。
- **作者出镜时（persona）**：只抓核心识别锚点，不追求面部细节复刻；年龄感偏 40 岁方向，成熟沉稳，不幼化；真实比例**长但不细**的长方椭圆脸、圆下巴，脸颊和下颌有横向宽度；**短直发**（顺直不卷）；大镜片浅灰眼镜；嘴唇整体偏厚、下唇尤其明显；**适中略垂眼型**，眼角可有极细微成熟纹理；**左右两腮 / 脸颊下半部与下颌线的极淡细微胡茬**；嘴周保持干净，嘴部只负责厚嘴唇识别；米色 T + 橄榄卡其短裤；无手表、不加深皱纹。

## 失败信号

出现以下情况，重生成或局部编辑：

- 左上角有「Workflow / 系统架构图 / 路线图 / 常见坑」等类型标题。
- 主角色像吉祥物、表情包或可爱卡通。
- **主角色 3D 漂移**：主色身体出现渐变/高光/体积建模/软陶/盲盒/Pixar 质感，不像 flat 2D 贴纸；轻微橙色明暗变化也算漂移。
- **主角色肢体错误**：同侧双手、重复手臂、手臂分叉、连线误画成第三只手或腿数不对。
- 主角色只是站在旁边，没有参与结构动作。
- 多个主角色像复制出来的同一素材，只是换了位置；没有不同姿态或动作分工。
- **多人形体比例漂移**：同一画面胶囊宽高比不一致（典型不合格：A 极瘦高、B 极矮胖、C 横扁）→ 直接不合格。
- **眼睛 / 手臂 / 腿不合格**：眼距/眼法批内不一致；臂/腿断线悬浮、同侧双手、分叉指、绳线粘连成第三只手、锚点错误（头顶/体中横腿）。
- **小石头有嘴**：胶囊上出现嘴线/牙/唇——直接不合格。
- **四肢批内漂移**：同图/同批线宽、锚点高度、末端画法不一致。
- **老杨特征漂移**：眼镜框型/发长/唇厚/脸型/穿搭批内不一；P1–P6 任一项 FAIL。
- **老杨表情越界**：夸张笑、露齿、背身无脸、仅气泡、幼态/叔感极端——见 `persona-author-identity.md` 表情稳定域。
- **跨图漂移**（同批 2 张及以上）：老杨眼镜框/发型/穿搭/脸型不一致；小石头胶囊比例或眼法不一致；一张有嘴/漩涡眼一张白圆眼 → 全批返修。
- 画面像 PPT、课程课件、正式流程图或复杂架构图。
- 元素太多、箭头太多、节点太多。
- 文字变成大段解释。
- 背景有纸纹、阴影、渐变、米色或噪点。
- 画面第一眼像实物摄影棚小场景，而不是白纸上的手绘草图。
- 中文错字严重或标注不可读。
- 画面把参考文件名、素材路径、内部资产名或生成说明当成正文内容。
- 画面太死板，没有怪诞隐喻。
- **persona识别丢失**：发型画卷/画簇（应为短直发）；脸型比例失真（应为长但不细的长方椭圆、圆下巴）；脸画成窄长、清瘦细脸或下颌过窄；眼睛画大或眼周过于年轻光滑；缺少眼角极细微成熟纹理；嘴唇画薄、嘴过小；**该有两腮胡茬却全净，或把胡茬画成嘴周胡子 / 八字胡 / 山羊胡 / 络腮胡**；其余核心识别锚点丢失。
- **persona背身/无脸**：仅「老杨讲」气泡、背对观众、后脑勺、侧背剪影，看不到眼镜与短发 → 不合格（知识卡/手绘/PPT 须正面或 3/4 可见面部）。
- **persona像老太太/老妇人**：灰卷白发、盘发、老年女性轮廓、与 spec 性别年龄不符 → 直接不合格。
- **persona手绘显嫩或显老**：脸窄、尖下巴、细脖子、娃娃脸、二十多岁光滑幼态脸、偶像化年轻脸会显嫩；深法令纹、眼袋、皱纹线会显老。返修时把年龄感调向偏 40 岁的中国男性成熟感，把脸改成长但不细，补足脸颊与下颌横向宽度，改圆润下巴、加粗脖子、稳实体态、平眉略低，仍不加皱纹。
- **主角色被persona带偏成 3D**：persona出镜时主角色被画成灰色鹅卵石、3D 圆润卡通或任何非 `the profile primary color` flat 2D 贴纸形象；主角色必须始终是本节开头定义的扁平黑线手绘风格，无论persona是哪种渲染风格。

## 迭代方法

- 太普通：让主角色成为动作主体，加入一个奇怪但成立的隐喻。
- 太复杂：删节点，只保留一个动作和 3-5 个短标注。
- 太可爱：强调 serious、deadpan、not cute、not mascot。
- **3D 漂移**：强调 flat 2D sticker, solid flat the profile primary color fill, thin black line limbs, no clay/vinyl/Pixar, no gradient on body.
- 太 PPT：去掉标题、边框、整齐网格和过多箭头，改成手绘草图。
- 太像实物图：去掉真实物件光影和摄影棚表面，改成线稿结构。
- 文字错：优先局部编辑；错得多就重生成并减少标注数量。

## 交付判断

高质量手绘图应该让读者先觉得「有点怪」，然后 1 秒内看懂结构。

如果第一眼像教程页，而不是白纸上的怪诞产品草图，就不合格。

**交付前必须输出 Confirm Gate**（见 `common-character-lock.md`）；结论非 `CONFIRMED` 不得保存或对外展示。

## 收到用户返修意见时

先查 `common-qa-repair.md` 的用户反馈映射表，找到对应返修方向，再按其中的返修输出格式（本轮保留/本轮调整/返修提示词）给出结果，不要凭感觉现改或整套重来。
