# 模式决策对比表

**目标**：快速区分四种模式，避免误判。

---

## 一句话区分

| 模式 | 核心定位 | 典型用途 |
|------|---------|---------|
| 实物图 | 处境小现场，一个隐喻 | 正文配图、情绪共鸣、项目故事 |
| 手绘图 | 结构解释图，一个流程 | 工作流拆解、方法论、系统关系 |
| 知识卡 | 独立传播容器，能收藏 | 朋友圈传播、方法拆解、课程总览 |
| PPT 演讲 | 连续多页演讲页面 | 直播分享、课程课件、主题演讲 |

---

## 详细对比表

| 维度 | 实物图 | 手绘图 | 知识卡 | PPT 演讲 |
|------|--------|--------|--------|---------|
| **尺寸** | 16:9 | 16:9 | 3:4/4:5/9:16 | 16:9 |
| **方向** | 横版 | 横版 | 竖版（手机海报可 9:16） | 横版 |
| **张数** | 单张或多张正文配图 | 单张或多张解释图 | 单张独立传播 | 整套连续页面（通常 8-20 页） |
| **信息量** | 画面更空（1 物件 + 4 标签） | 讲清楚重点（1 结构 + 5-8 批注） | 内容更完整 / 完整海报 | 按页面类型变化 |
| **留白** | 很大，摄影棚感 | 较大，白板感 | 适中，有结构但不拥挤 | 按页面类型变化 |
| **标题** | 无正式标题 | 无正式标题 | 必须有标题 | 每页有标题 |
| **模块** | 无 | 无（只有核心结构） | 有步骤/模块/行动清单 | 有（按页面类型） |
| **阅读顺序** | 一眼看懂冲突 | 跟着结构箭头看 | 有明确阅读顺序（从上到下） | 逐页演讲顺序 |
| **传播用途** | 文章正文配图 | 文章正文配图 | 单独发朋友圈/小红书 | 演讲/直播/课程 |
| **核心区分** | 真实物件小现场 | 白板手绘结构 | 能收藏、值得存 | 连续多页、有节奏 |

---

## 典型用户输入 → 模式判断

| 用户说 | 判断 | 理由 |
|--------|------|------|
| "配 4 张图" | 实物图 or 手绘图 | 正文配图，看内容类型（处境 → 实物；流程 → 手绘） |
| "做一张能发朋友圈的方法论图" | 知识卡 | 关键词："能发" = 独立传播；"方法论" = 结构内容 |
| "画一张白板图解释工作流" | 手绘图 | 关键词："白板图" + "工作流" = 结构解释 |
| "做一套 10 页 PPT" | PPT 演讲 | 关键词："一套" + "10 页" = 连续多页 |
| "出一张能收藏的图" | 知识卡 | 关键词："收藏" = 独立传播价值 |
| "给这篇文章配图" | 实物图 or 手绘图 | 正文配图，看内容（情绪/处境 → 实物；流程/结构 → 手绘） |
| "做一张 9:16 的图" | 知识卡（手机海报） | 竖版尺寸 = 知识卡模式 |
| "课程总览图" | 知识卡 or PPT | 单张 → 知识卡；多页 → PPT |
| "彩蛋长卷 / 项目复盘" | 实物图长卷 | 2.6:1-3:1 超横版，多节点故事 |

## 边界案例决策

### 案例 1："一页 PPT"还是"知识卡"？

| 用户输入 | 判断 | 理由 |
|---------|------|------|
| "做一页 PPT" | **知识卡** | 单张优先知识卡（竖版传播价值更高） |
| "做一页 PPT 风格的封面" | **PPT 演讲** | 明确说"PPT 风格" = 演讲页面美学 |
| "做一张能放 PPT 里的图" | **知识卡** | 实际是"独立传播图"，可插入 PPT 使用 |

**原则**：单张且追求传播/收藏 → 知识卡；连续多页演讲 → PPT。

---

### 案例 2："方法论图"是"手绘图"还是"知识卡"？

| 用户输入 | 判断 | 理由 |
|---------|------|------|
| "画一张图解释这个方法" | **手绘图** | 关键词："解释" = 认知拆解，16:9 留白 |
| "做一张方法拆解图，能发朋友圈" | **知识卡** | 关键词："能发" = 独立传播，竖版容器 |
| "给这篇方法论文章配图" | **手绘图** | "配图" = 正文配图，不是独立传播 |
| "把这个方法做成收藏图" | **知识卡** | 关键词："收藏" = 独立传播价值 |

**原则**：正文配图 + 解释 → 手绘图；独立传播 + 收藏 → 知识卡。

---

### 案例 3："实物图"还是"手绘图"？

| 内容类型 | 判断 | 理由 |
|---------|------|------|
| 用户痛点、处境、情绪 | **实物图** | 需要真实物件隐喻，引发共鸣 |
| 流程、结构、系统关系 | **手绘图** | 需要箭头、模块、连接线 |
| 会议被拉回、需求返工 | **实物图** | 处境类，选母版 01/04 |
| Agent 工作流、RAG 流程 | **手绘图** | 流程类，选 Workflow 结构 |
| 项目故事、个人经历 | **实物图（长卷）** | 多节点叙事，2.6:1-3:1 |

**原则**：问"是处境还是结构"——处境选实物，结构选手绘。

---

## 升级模式决策（实物/手绘 → 知识卡/PPT）

| 场景 | 是否升级 | 理由 |
|------|---------|------|
| 用户说"配图"，内容是步骤 | ❌ 不升级，先推荐实物/手绘 | 用户未明确要独立传播 |
| 用户说"能发出去的图" | ✅ 升级到知识卡 | 明确独立传播需求 |
| 用户说"做一套课件" | ✅ 升级到 PPT | 明确多页演讲需求 |
| 内容是 2000 字长文方法论 | ⚠️ 先问用户 | 可以是"多张手绘图"或"1 张知识卡" |

**原则**：不主动升级，除非用户明确要"独立传播"或"整套演讲"。

## 快速决策流程图

```text
用户输入
    ↓
显式风格词？
    ├─ 是 → 按风格词判断（实物图/手绘图/知识卡/PPT）
    └─ 否 ↓
是否明确说"多页/整套/课件"？
    ├─ 是 → PPT 演讲模式
    └─ 否 ↓
是否明确说"能发/收藏/传播/朋友圈"？
    ├─ 是 → 知识卡模式
    └─ 否 ↓
内容是"处境/情绪/痛点"？
    ├─ 是 → 实物图模式
    └─ 否 ↓
内容是"流程/结构/方法论"？
    ├─ 是 → 手绘图模式
    └─ 否 ↓
不确定 → 先给 1-2 句说明并选实物/手绘，不反复追问
```

## 常见错误

| 错误 | 正确做法 |
|------|---------|
| 用户说"配图"就用知识卡 | ❌ "配图"默认正文配图（实物/手绘），不是独立传播 |
| 用户说"方法论"就用知识卡 | ❌ 先看是"解释"（手绘）还是"传播"（知识卡） |
| 用户说"一页 PPT"就用 PPT 模式 | ❌ 单张优先知识卡，除非明确"PPT 风格" |
| 把手绘图信息量加到"完整海报" | ❌ 手绘图只用"画面更空""讲清楚重点"，需要更多信息切到知识卡 |
| 把知识卡做成 16:9 | ❌ 知识卡必须竖版（3:4/4:5/9:16） |
