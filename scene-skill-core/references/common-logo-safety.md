# 通用 Logo 与品牌资产安全规则

本文件只描述通用安全原则。具体品牌边界、私有资产目录和工牌细节由当前 `ip-profiles/<ip-id>/logo-safety.md` 定义。

## 默认原则

- 通用 Skill 流程不主动绑定任何公司、商标或真实 Logo。
- 公开包不应分发真实品牌 Logo 源文件。
- 用户没有明确触发 Logo / 工牌 / 展会 / 品牌物料时，生成图里不要自动加入真实 Logo。
- 需要真实 Logo 时，必须由用户本地私有提供，并由用户自行确认授权。
- 内部 VI、Logo 源文件、PPT 模板、字体包和品牌规范文档只可作为本地私有参考，不得写入公开 README、examples、profile 文档或生成提示词。
- 公司简介、使命愿景、slogan、对外宣传版本等正式文案也属于私有品牌材料；未经用户明确确认可公开使用时，只能转译为去品牌化的场景规则。

## 使用方式

- 生图阶段优先预留空白 Logo 区，不让模型凭空重画 Logo。
- 用户要求 Logo 完全一致时，交付前应使用授权原图本地贴入。
- 不要近似仿画、改造几何、重绘第三方官方标识。
- 不要生成真实姓名、手机号、工号、二维码、部门架构或未经提供的履历信息。
- 不要暴露私有飞书链接、内部资料标题、源文件名或下载包名称；公开文档只保留去品牌化的规则描述。
- 不要把正式公司介绍、使命、愿景或 slogan 原文写入公开示例图、README、prompt 模板或默认 profile。

## 主角色边界

无论当前 profile 的主角色是什么，真实 Logo 都不得直接印在主角色身体、眼睛、皮肤或默认服装上。Logo 只能作为明确触发后的场景道具元素，例如工牌、展板、物料、贴纸。

## Profile 责任

每个 IP profile 如需支持品牌露出，应在自己的 `logo-safety.md` 中声明：

- 哪些触发词会启用 Logo / 工牌 / 展会物料。
- 私有 Logo 应放在哪个 git ignored 目录。
- 主角色、persona、道具各自是否允许佩戴或展示 Logo。
- 公开示例是否允许出现品牌元素。
