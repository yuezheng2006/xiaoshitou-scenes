# 母版选择与拦截协议

## 母版 PNG 状态

标准实物图母版文件名为 `assets/masters/01-alignment-pull-in.png` … `assets/masters/06-filter-loss.png`。

1. 标准 16:9 实物图必须锁定母版**类型**（01–06）并填写锁定字段。
2. 生成前 Read 对应 PNG + 本文档抽象骨架 + `physical-style-dna.md` + `current profile character assets` 参考图。
3. 彩蛋长卷没有公开 PNG 母版；使用本文档与 `physical-style-dna.md` 的长卷文字骨架。
4. 若标准 PNG 缺失，以本文档各节**抽象骨架** + `physical-style-dna.md` 比例/留白 + `current profile character assets` 主角色参考图为质量锚点。

下文「目标画面」描述的是母版类型的**目标规格**（供重生成 PNG 与无 PNG 时对齐），不是可机械复制的布局。

本文件用于修复两个关键风险：

- 元素清单化：模型把主题词、物件和标签当成清单堆进画面。
- 母版复刻化：模型把高质量案例当成可描摹模板，直接复制旧道具、挡板、标签堆等空间拓扑。

正确做法是先选择一个案例作为**质量锚点**，再抽取它的抽象不变量，最后为当前内容写出必须变异的地方。母版不是可复制画面，也不是空间模板；它只提供质量标尺和动作类型。

## 母版使用总原则

每次使用母版必须同时满足：

- **保留不变量**：比例克制、留白、真实物件质感、主角色参与核心动作、标签少而准；**同批配图还须保留 Character Lock（同一主角色）**。
- **必须变异**：主物件、空间方向、动作关系、道具组合、标签位置、视角或叙事重心中，至少 3 项必须和母版不同。
- **先读懂再好看**：不看说明，读者 3 秒内也能说出“主角色正在被什么困住 / 挡住 / 拉扯 / 标注 / 分层 / 拦截”。

如果生成图第一眼像某张案例的换皮版，就判失败，即使它背景、标签和主角色都对。

## 标准模式母版索引

### 01 alignment-pull-in

文件：

```text
assets/masters/01-alignment-pull-in.png
```

目标画面（母版类型规格，不是可复制布局）：

- 左侧主角色用力拉黑色挂绳 / 弹力绳。
- 右侧黑色风琴文件夹 / 档案夹被绳勒住，夹内露出白纸。
- 标签贴在文件夹正面（如「对齐一下」「再开一次」「谁拉的」）。
- 纯白底，配件极少，留白大。

适合：

- 会议、同步、对齐、临时拉人、需求被再次打开、下班前被叫回。

抽象骨架：

- 多个外部请求从主物件一侧施加拉力。
- 主角色被拉向任务入口，或正在把某物「再拉开 / 再对齐」。
- 留白很大，配件很少。

可复用关系：

- 「任务 / 会议 / 需求」作为被拉开的物件或入口。
- 主角色与挂绳、胶带、文件夹、门把手发生拉力。
- 标签在来源或主物件附近，不要每个物件都贴满字。

失败信号：

- 直接复制「左主角色拉绳 + 中/右文件夹 + 标签贴夹面」的拓扑。
- 主物件过大，变成商品特写。
- 线太多，像蜘蛛网。
- 标签和消息卡太多，像信息流合集。
- 误画成电脑 + 会议卡片旧版 generic 拓扑（本仓库母版已是文件夹方案）。

### 02 info-overload

文件：

```text
assets/masters/02-info-overload.png
```

目标画面（母版类型规格，不是可复制布局）：

- 左侧一个现代消息收纳盒 / 任务队列盒持续吐出小卡片、任务条或消息标签。
- 主角色用透明资料托、轻薄挡板或双手挡在涌出物前方。
- 可选 1 个很短的来源标签，但不是 UI 截图，也不要出现真实品牌名。
- 标签点出过载话术（如「消息太多」「马上回」「谁点的」）。

适合：

- 消息过载、点歌条涌出、任务涌出、已读不等于完成、催促。

抽象骨架：

- 一个真实物件作为「输出口 / 涌出口 / 源头」。
- 主角色正在挡、推、接、压住或整理涌出的东西。
- 画面表达「东西出来太多，人挡不住」。

可复用关系：

- 主物件是「出口」，不是信息展示屏；出口可以是消息收纳盒、任务队列盒、智能提醒牌、桌面小屏边缘或轻量服务队列装置，不必是手机。
- 主角色必须挡、推、接或整理，不是站着看。
- 标签只点出 2-4 个真实话术。

失败信号：

- 直接复制「左侧出口 + 中间卡片流 + 主角色挡板 + 右上角吊牌」的拓扑。
- 当前内容不是消息 / 任务 / 涌出经验，却仍然强行使用输出口结构。
- 出现聊天 UI 或截图。
- 消息卡文字太多、标签过密。
- 主物件过大，画面像设备广告。
- 误画成旧版手机消息堆、旧办公出纸或怀旧道具拓扑。

### 03 problem-knot-alert

文件：

```text
assets/masters/03-problem-knot-alert.png
```

目标画面（母版类型规格，不是可复制布局）：

- 黑色粗线缆横穿画面，中央打成结。
- 主角色双手握住结附近，正在排查 / 解结。
- 标签可竖排在结旁（如「又断了」「小问题」「先排查」）。
- 纯白底，无房间或行业实景背景。

适合：

- 报警、返工、打结、问题回流、边界情况、设备线缆故障、下班失败被拽回。

抽象骨架：

- 一个真实长条物或路径形成卡点、结、回流或缠绕。
- 主角色的动作集中在卡点上。
- 画面表达「本来应该顺着走，但在这里卡住 / 回来了」。

可复用关系：

- 线缆、软管、挂绳就是冲突的物理路径。
- 主角色动作集中在「结」上。
- 标签贴近问题点，不要把全流程画出来。

失败信号：

- 直接复制「横穿黑线缆 + 中央结 + 主角色双手握结 + 右侧竖排标签」的拓扑。
- 线缆变成装饰曲线，和主角色没有力学关系。
- 标签散太远，看起来像海报注释。
- 配件过多，冲淡「打结」的核心现场。
- 误画成桌游卡牌、解谜 MASTER 卡（品牌误认）。

### 04 review-rework

文件：

```text
assets/masters/04-review-rework.png
```

目标画面（母版类型规格，不是可复制布局）：

- 中央轻薄评审卡 / 透明标注片 / 验收清单板，上面有红圈、红线下划线或状态贴。
- 主角色拿批注笔或标签机，正在圈改、贴验收贴纸或对齐标注。
- 左侧可有圆角评审卡或状态贴（如「再改改」），左侧可有气泡短标签。
- 表达「表面完成了，还要被复核 / 改稿」。

适合：

- 审查、校对、复核、返工、验收、谁来兜底。

抽象骨架：

- 一个真实评审工具把内容局部框住、标注或压住。
- 主角色在检查区域里执行修、查、改、贴、圈。
- 画面表达「表面完成了，但还要被复核 / 兜底」。

可复用关系：

- 批注笔、透明标注片、验收贴纸、圆角评审卡是「被审查 / 要再看」的物理工具。
- 主角色必须在被检查或检查别人之间形成清楚关系。
- 标签要像审查意见，不像流程名称。

失败信号：

- 直接复制「中央评审卡 + 主角色圈改 + 左侧状态贴 + 左侧气泡标签」的拓扑。
- 标注工具太大，主体变成单一物件特写。
- 主角色只是站在标注区域里，没在做动作。
- 卡片太满，像办公素材拼贴。

### 05 automation-relabel

文件：

```text
assets/masters/05-automation-relabel.png
```

目标画面（母版类型规格，不是可复制布局）：

- 大尺寸挂绳工牌 / 员工工牌 / 服务身份牌，牌面有多色标签块（如「自动化」「AI 先做」「人来兜底」「谁改的」）。
- 主角色一手扶工牌，一手贴状态贴 / 调整标签条。
- 身份牌是**叙事道具**（岗位 / 身份 / 权限 / 服务值班），不是品牌海报。
- 普通身份牌默认不要印真实 Logo、公司名或商标；若用户明确要求员工工牌 / 服务工牌 / 值班牌，则按 `common-logo-safety.md` 预留 Logo 区或使用用户本地私有授权 Logo，默认不主动启用。
- 横向铺开，配件极少。

适合：

- AI 先做、人来兜底、岗位重组、自动化牵引、身份被重新贴标签。
- 员工工牌、K 歌服务值班、现场权限、服务工单流转。

抽象骨架：

- 一个真实标签、身份牌或纸带正在重新定义某个东西。
- 主角色用身体阻止、拉回、撕开或按住这个标签变化。
- 画面表达「系统给了新名字，但人还在承担后果」。

可复用关系：

- 标签 / 纸带 / 工牌代表系统给人的新定义，或员工在服务现场承担的身份与权限。
- 主角色和挂绳 / 标签发生拉扯、贴状态或调整标签。
- 员工工牌系列中，Logo 只在工牌牌面，不能出现在主角色身上。
- 主体横向铺开，配件极少。

失败信号：

- 直接复制「左主角色 + 中挂绳工牌 + 改字工具 + 多色牌面标签」的拓扑。
- 做成 AI 工具广告、机器人图或胸口 Logo 胸标。
- 写太多「自动化 / 岗位 / 重组」概念词。
- 主角色站在旁边，无法看出它在抵抗或兜底。
- 在非办公 / 权限 / 服务身份场景误加工牌。
- 品牌 Logo 画错、异色、贴到主角色身上，或生成真实员工姓名/工号/二维码。

### 06 filter-loss

文件：

```text
assets/masters/06-filter-loss.png
```

适合：

- 筛选、过滤、关键词、用户分层、机会被分层或拦截。

抽象骨架：

- 一个智能筛选板、标签队列、权限闸口或自动化流转槽制造入口和出口。
- 主角色和被分层卡片发生接触：推入、接住、救回或卡住。
- 画面表达“还没真正被看见，就先被规则分层或拦截”。

可复用关系：

- 筛选板 / 权限闸口 / 自动化流转槽是主物件，其他元素都服务“被分层 / 被拦截”。
- 主角色动作要接触卡片、队列或闸口。
- 颜色只做几个小节奏点。

失败信号：

- 直接复制“中央传统过滤器 + 下方堆叠材料 + 左侧主角色递卡”的拓扑。
- 筛选板之外又加多个主物件。
- 画成流程图或招聘系统 UI。
- 标签解释过度，失去情绪按钮感。

## 必填母版锁定字段

每张标准实物图在生成前必须写出：

```text
母版类型（01–06）：
Character Lock（2 张及以上）：
抽取的不变量：
当前内容的变异点：
画面 3 秒读懂句：
当前内容适配：
要避免的失败信号：
```

这些字段任何一项说不清，不生成，先重写 shot list。

变异点至少写 3 个，例如：

- 换掉主物件类别。
- 改变左右/中心空间关系。
- 改变主角色动作，不再做同一种推/拉/挡。
- 删除母版里的标志性配件。
- 改变标签出现方式。
- 改变视角或物件姿态。

## 出图拦截规则

生成后的第一张图永远只是候选图，不是交付图。必须和所选母版做视觉比对：

- 主体比例是否接近母版，而不是商品特写或缩略摆件。
- 留白是否接近母版，而不是元素铺满。
- 真实物件是否像同一张摄影棚照片。
- 主角色是否承担母版级的物理动作。
- 标签是否像情绪按钮，而不是概念说明。
- 是否出现“主题元素清单化”：把电脑、手机、卡片、线、标题都堆上去。
- 是否出现“母版复刻化”：主体、方向、配件、主角色动作和标签位置都像旧案例换字。
- 是否 3 秒读懂：不看解释，读者是否能说出画面冲突。

只要出现“元素清单化”“母版复刻化”“不像母版正文配图”或“看不懂冲突”，就必须重写提示词并重生成，不得交付为最终图。
