# 通用 QA 辅助：装饰性测试 / 反馈映射 / 返修输出格式

本文件是 `physical-qa-checklist.md`、`handdrawn-qa-checklist.md`（以及后续知识卡模式、PPT 演讲模式的 QA）共用的辅助判断工具，不重复各模式已有的专属检查项，只补三块通用能力。

## 装饰性测试（角色是否只是站在旁边）

每张图 QA 时都做一次这个测试：

```text
如果把主角色（以及出镜的persona）从画面里去掉，这张图讲的道理是否还完全成立？
```

- 如果答案是「完全成立」，说明角色只是装饰，没有真正参与核心动作——判定不合格，需要返修：让角色直接承担挡/拉/推/扛/解/贴/塞/接/修/守等物理动作，或在手绘图里承担画箭头、递模块、指向结构等概念动作。
- 如果答案是「不成立，去掉角色画面就讲不清楚了」，说明角色参与了核心叙事，可以继续走后续检查项。

persona出镜时单独判断一次：去掉persona，画面核心冲突是否仍然成立；如果persona只是并排站立的背景装饰，同样判定不合格。

## 用户反馈映射表

收到用户的主观反馈后，先在这张表里找对应的具体返修方向，不要凭感觉现改：

| 用户反馈 | 返修方向 |
| --- | --- |
| 不像主角色 | 对照当前 profile 的 `character.md` 和主角色锚点资产，强化名称、主色、形体、眼睛、肢体和维度锁；检查是否漂成非 profile 风格 |
| 不像persona / 不像我 | 实物图对照 `author-persona-spec.png`（复杂动作/小比例加 `author-persona-actions.png`）；手绘图/知识卡/PPT 同时对照 `author-persona-spec.png`（身份锚点）+ `author-persona-handdrawn.png`（渲染语言），强化发际线、脸型、眼周、嘴唇、体态；脸型必须是长但不细的长方椭圆，保留脸颊和下颌横向宽度；年龄感偏 40 岁方向，眼角有极细微成熟纹理，嘴唇整体偏厚、下唇尤其明显；查 `common-persona-routing.md` 小比例最低识别线 |
| 预览/总览图老杨不像 / 粗黑框 / 黑长袖 | 文档图也必须走 Persona Identity Lock；过小格子改抽象符号；对照 spec 六项识别返修，禁止 generic 讲师 |
| 老杨背身 / 后脑勺 / 像老太太 | 知识卡/手绘/PPT 须正面或 3/4 可见眼镜+短发+厚下唇；禁止灰卷白发、老年女性轮廓、仅「老杨讲」气泡无脸；传 spec + handdrawn 返修 |
| 同批两张不像 / 小石头眼法不一 | 声明跨图一致锁；全批同一胶囊比例+白圆眼画法；同图多人只改缩放/姿态/分工，禁止换形体或漩涡眼/有嘴 |
| 太可爱 / 太萌 | 去掉夸张表情和萌化比例，强化「认真、略笨、有点倔」的气质，改用更克制的动作 |
| 太普通，没记忆点 | 换一个更具体的真实物件或概念隐喻，让主角色的动作更具体、更有物理冲突 |
| 太丑 / 太粗糙 | 检查线条质量、比例和光影是否统一，保持简笔但不潦草 |
| 太复杂 / 信息太多 | 删减物件、标签或模块，只保留一个核心冲突/结构，回到硬性预算 |
| 太空 / 信息太少 | 补 1-2 个短标签或一个次要元素，但不要突破硬性预算上限 |
| 颜色不喜欢 | 保持白底 + 橙/黑主色，只调整点缀色（红/橙/蓝），不要整体换色系 |
| 角色像装饰，没参与 | 做「装饰性测试」，给角色分配明确的物理或概念动作 |
| 太像 PPT / 太商业 | 去掉标题框、整齐网格、大段说明文字，回到留白手绘/简笔实物质感 |
| 太像旧案例换皮 | 重新从当前内容找隐喻，至少换 3 个变异点（物件/动作/标签/视角） |
| 主角色 3D 漂移 | 强制回到 flat 2D：纯色平涂、无渐变高光、细黑线四肢、平面白圆眼 |
| persona被带成 3D 灰石头 / 主角色被带偏 | 重申 2D Flat Lock：主角色始终 flat 2D，即使persona是成熟克制的风格化 3D，两者渲染语言必须区分 |
| 文字错 / 看不清 | 优先局部编辑重写文字；错得多就整张重生成并减少标签数量 |
| 多人形体不一样 / A 瘦 B 胖 | 强制形体一致锁：全批相同胶囊宽高比，只允许整体等比缩放 ±10–15%；重写 prompt 禁止 fat/thin/wide 变异词 |
| 眼睛不对 / 眼距怪 / 不像 ref | 对照 reference 重写 Eyes Lock：两只白圆、直径 12–18% 体宽、间距 1.2–1.8 眼径；批内一致 |
| 手臂断 / 多手 / 腿锚点错 | 重写 Limbs Lock：2 臂 2 腿；臂从体侧上 1/3、腿从底缘连续；数线条做 L1–L4 快检后重生成 |
| 小石头有嘴 / 用嘴表达情绪 | 加 Negative: no mouth on capsule；情绪改汗滴/倾斜/动作；Confirm Gate E1 FAIL 返修 |
| 四肢批内线宽/锚点不一 | 全批统一 Limbs Lock 文案；跨图稳定 FAIL 则整批重生成 |
| 老杨表情不对 / 露齿 / 背身 | 重写 Persona Expression Lock；须正面或 3/4；P6–P7 逐项对照 spec 返修 |
| 老杨批内特征漂移 | 同批共用 Persona Feature Stability Lock；禁止一张粗框一张细框、一张背身一张正脸 |

如果收到的反馈不在表里，先按最接近的一行类推，再补充新行到这张表，方便下次直接复用。

## 返修输出格式

给用户返修说明时，统一用这个结构，不要写成散文：

```text
本轮保留：
- [角色锚点：primary-character-reference.png / author-persona 系列]
- [母版类型或结构类型]
- [核心物件/核心结构]
- [已确认的短标签/文案]

本轮调整：
- [具体改动 1]
- [具体改动 2]

返修提示词：
[可直接使用的中文优先完整提示词，包含上面保留项和调整项；必要时保留少量英文风格关键词]
```

不要因为一次返修就重新发明整套角色或整套构图，除非用户明确说「重来 / 角色方向不对」。
