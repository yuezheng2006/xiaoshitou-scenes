# 通用角色锁规则

本文件只描述通用流程；**具体角色定义的单一真相来源**：

- **小石头规则** → `ip-profiles/default-little-stone/character.md`（唯一定义）
- **老杨规则** → `ip-profiles/default-little-stone/persona-author.md` 及其分层文件（唯一定义）
- **本文件职责**：只描述"如何锁定""如何检查""跨图稳定机制"，不重复定义角色本身

## 读取顺序

1. 先读取当前 profile 的 `profile.md`。
2. 再读取 profile 指向的 `character.md`（小石头唯一定义；`{IP_DESC}` / `{IP_STYLE_ADAPT}`；默认单锚点）。
3. 触发 persona 时，读取 `common-persona-calibration.md` + `persona-author.md` + `persona-author-assets.md`（**老杨双参考 · 对齐人**）+ `persona-quick-checklist.md`。
4. 生图前：小石头传设定图；老杨出镜传对应模式全景并完成参考图校准。组装顺序见 `common-prompt-slots.md`。
5. 多人、复杂姿态或小比例远景时，再读取对应动作扩展资产。
6. 用户要求无角色 / 纯物件时，改读 `ip-profiles/none/`，跳过本文件对默认角色的强制项。

## Character Lock

一次生成 2 张及以上时，必须为当前 profile 主角色声明全批 Character Lock：

- 固定角色名、主色、基本形体、眼睛、肢体风格和禁忌。
- 每张可变化姿势、动作、道具、标签和相对站位。
- 全批候选图必须像同一个 IP 家族，不得漂成不同 mascot。

## 维度锁

如果当前 profile 声明主角色是 flat 2D（如默认的小石头），则每张图都必须写入 2D Flat Lock，并在 QA 时检查：

- 主角色身体是否仍为平涂色块。
- 眼睛和肢体是否仍是 profile 指定的平面风格。
- 肢体数量是否正确：每个主角色只能有两只手臂、两条腿；不得出现重复手臂、分叉手、额外手指状线条，牵引线/绳子/箭头也不能被画成第三只手。
- 是否被真实物件、3D persona 或模型默认风格带成 3D。

**维度锁具体内容**：见 `character.md` 的 “2D Flat Lock + Limbs Lock” 章节（唯一定义），本文件不重复。

如果未来 profile 声明主角色为 3D、线稿或其他风格，也应在 profile 内给出对应维度锁；通用层只负责要求”生成前声明、生成后核对”。

## Persona Identity Lock（画面出现 persona 肖像时强制）

只要画面里会出现 persona 的**脸、半身、全身或可被读成该 persona 的小比例人形**（含双 IP、文档预览、能力总览、README/飞书示例图），生成前必须写入 Persona Identity Lock。

**Persona Lock 具体内容**：见 `persona-quick-checklist.md`（快速决策版）或 `persona-author-identity.md`（完整规范版），本文件不重复。

**触发后必读**：`persona-quick-checklist.md` 第四步"生成前并列写入"。

**文档 / 预览 / 能力总览 / 仓库示例图** 同样适用，不得因「不是正文配图」跳过 Persona Lock。若格子太小、只能放符号：

- 优先用 **抽象符号**（双 IP 箭头、职责标签、剪影图标），**不要**画一张未锁定的 generic 人脸；
- 若必须出现肖像，仍走完整 Persona Lock + 资产传图，生成后仍做形象检查。

未触发 persona 且画面不应出现 persona 时，禁止偷塞 persona 肖像。

## 四肢锁定（Limbs Lock · 生成前必写）

**Limbs Lock 具体内容**：见 `character.md` 的 "2D Flat Lock + Limbs Lock" 章节（唯一定义），本文件不重复完整 prompt 片段。

**生成前必做**：每张 prompt 必须并列写入 Limbs Lock（计数 + 臂锚点 + 腿锚点 + 线质 + 末端 + 无嘴）。

**核心要点**：
- 每个体恰好 2 臂 + 2 腿
- 臂从体侧上 1/3、腿从底缘连续向下
- 细黑线，禁止 3D 圆管
- 无嘴（情绪靠动作+汗滴）

多人 / 同批图：四肢 **线宽、锚点高度带、末端画法** 全批一致；差异只来自动作姿态，不换线质或锚点规则。

## Persona 特征稳定锁 + 表情锁（生成前必写）

画面出现老杨（或当前 profile persona）时，除 Persona Identity Lock 外，**必须并列写入** Feature Stability Lock 和 Expression Lock。

**具体内容**：见 `persona-quick-checklist.md` 第四步（快速决策版）或 `persona-author-identity.md`（完整规范版），本文件不重复完整 prompt 片段。

**核心要点**：
- 脸型：长但不细的长方椭圆
- 眼镜：大镜片浅灰细框（非粗黑框）
- 发型：短直发 3-5cm（非卷发/灰白发）
- 嘴唇：厚下唇
- 穿搭：米色短袖 T + 橄榄卡其短裤（同批禁止换装）
- 配件层：眼镜（face）+ 短发（head）场景/表情变化时必须保留
- 可见性：知识卡/手绘/PPT 须正面或 3/4（禁止背身）
- 表情：选用 Expression Preset E0–E4（见 `persona-author-identity.md`），唇闭合或微解释不露齿
- 多场景：先 1 张预览门禁；same exact person；跨图脸部差异要小；FAIL 只返修单张
- 禁止每张换脸 / 丢眼镜 / 未预览就批跑

## 形象检查（生成后强制，与模式 QA 并列）

每张候选图交付前，除模式 QA 外，必须做 **形象快检**（主角色 + persona，若出现），并汇总 **交付确认度**（见下文 Confirm Gate）：

| 检查项 | 主角色 | persona（若出现） |
| --- | --- | --- |
| 锚点对照 | primary-character-reference.png（单锚点） | 实物：实体 panorama；手绘：手绘 panorama + 实体 panorama |
| 维度锁 | flat 2D 平涂、无渐变/高光 | 渲染语言与模式一致，非泛化头像 |
| 眼睛 | 白圆眼+可选 tiny 瞳孔；位置/大小/间距批内一致 | 适中略垂眼；非 anime 大眼 |
| 四肢 | Limbs Lock 四点检（见下） | 人体比例自然；手参与递/指/画 |
| 六项识别 | 胶囊形、无嘴 | 眼镜/发型/穿搭/唇形/肤色/成熟感 |
| 表情 | 无嘴；靠动作/汗滴 | 表情稳定域内；正面或 3/4 |
| 失败处理 | 不交付，重写 prompt 或重生成 | 不交付，对照 spec 禁止偏移区返修 |

人像参考校准失败或形象检查失败 = CRITICAL，不得标 DONE。文档/预览/飞书/仓库示例图与正文配图同一标准。

## 交付确认度（Confirm Gate · 生成后必输出）

**对 Agent**：候选图永远只是候选；标 DONE / 写入 `assets/examples/` / 插入飞书前，必须逐项核对。任一 **Tier 1 CRITICAL** 项 FAIL → **REJECT**，列出失败项，不得交付。

**对用户**：隐藏技术术语（L1/P3 等代号），只在 FAIL 时用自然语言说明具体问题。

### 三级检查系统（P1.1 简化）

**检查分级原则**：
- **Tier 1: CRITICAL** - 影响 IP 核心识别，必须通过
- **Tier 2: IMPORTANT** - 影响整体质量，尽量满足
- **Tier 3: NICE-TO-HAVE** - 追求卓越，建议但不强制

### 交付标准

✅ **所有 Tier 1 通过** → 可交付（标注 Tier 2/3 问题供用户选择）
⚠️ **部分 Tier 2 失败** → 可交付但建议优化
❌ **任一 Tier 1 失败** → 必须返修

**对用户说明示例**（FAIL 时）：
```
❌ "这张图小石头多了一只手臂，需要重新生成"（Tier 1 FAIL）
   而不是："形象 Confirm Gate: L1 FAIL (extra arm)"

⚠️ "这张图整体不错，小石头的手臂锚点略低（建议从体侧上 1/3 位置伸出），是否需要优化？"（Tier 2 问题）
   
✅ "这张图的形象检查通过，小石头和老杨的特征都符合标准"（Tier 1 全过）
   而不是："形象 Confirm Gate: CONFIRMED"
```

### 小石头检查清单（分级）

#### Tier 1: CRITICAL（必须通过）

| # | 检查 | PASS 条件 | 容忍度 | 典型 FAIL |
| --- | --- | --- | --- | --- |
| **L1** | **计数** | 每个体恰好 2 臂 + 2 腿 | 无容忍 | 同侧双手、第三只手、腿数错 |
| **E1** | **眼睛** | 两只白圆眼 | 直径 10-20% 体宽 | anime 大眼、单眼、3D 反光眼 |
| **E2** | **无嘴** | 胶囊上无嘴线/牙/唇形 | 无容忍 | 有嘴线、笑脸、表情符号 |

#### Tier 2: IMPORTANT（尽量满足）

| # | 检查 | PASS 条件 | 容忍度 | 典型问题 |
| --- | --- | --- | --- | --- |
| **L2** | **锚点** | 臂从体侧上 1/3、腿从底缘 | 臂：上 1/4-2/5（25-40%） | 断线悬浮、锚点过低/过高 |
| **L3** | **线质** | 细黑线、与 ref 同级线宽 | ±20% 线宽波动 | 粗臂、3D 圆管、批内不一 |
| **L4** | **末端** | 每臂 ≤1 个简化握持 | 小圆/C形/线钩 | 分叉指、额外小手、绳线粘连 |

#### Tier 3: NICE-TO-HAVE（建议优化）

| # | 检查 | 理想值 | 可接受范围 |
| --- | --- | --- | --- |
| **L5** | **臂长** | 约 0.6-0.9 体高 | 0.5-1.0 体高（功能正常） |
| **L6** | **腿长** | 1-2 段细黑线 | 视觉协调即可 |
| **E3** | **眼距** | 1.2-1.8 眼径 | 1.0-2.0 眼径（视觉协调） |
| **E4** | **形体比例** | 宽高比 1:1.3-1:1.6 | 1:1.2-1:1.7（竖椭圆感） |

### 同批 / 跨图稳定（2 张及以上）

- 小石头：胶囊比例、眼法、四肢线宽、**无嘴** 全批一致
- 老杨：眼镜框型、发长、唇厚感、脸型、穿搭全批一致；表情仅在稳定域内变化

### 确认输出格式（内部记录，不展示给用户）

```text
形象快检（内部）：
- 小石头 L1-L2（计数+锚点）：PASS / FAIL（失败项：...）
- 小石头 E1（眼睛）：PASS / FAIL（失败项：...）
- 老杨 P1/P3/P6（核心识别）：PASS / N/A / FAIL（失败项：...）
- 老杨 P12 身高比例（全身/大半身 CRITICAL）：PASS / N/A / FAIL（失败项：...）
- 跨图稳定：PASS / N/A / FAIL（失败项：...）
内部结论：CONFIRMED 可交付 | REJECT 需返修
```

**只有结论为 CONFIRMED** 才能标 DONE、保存示例、更新飞书。FAIL 时按 `common-qa-repair.md` 的"问题-原因-方案"格式对用户说明，并 **重写 Limbs Lock / Persona Expression Lock** 后再生成。

## 跨图 / 同批一致锁（一次出 2 张及以上时）

同一次任务、同一系列（如飞书示例 #17+#18、gallery 成套样张、批量正文配图）必须**共用一套** Character Lock + Persona Identity Lock，并在生成后做**跨图快检**：

| 对象 | 全批必须一致 | 允许变化 |
| --- | --- | --- |
| 小石头 | `#f39800` 竖胶囊宽高比、白圆眼+瞳孔画法、细线肢线宽、无嘴/无 3D | 整体缩放 ±10–15%、倾斜、朝向、站位、动作、道具 |
| 老杨 | 线稿/3D 渲染语言、眼镜（大镜片浅灰细框）、短发 3–5cm、米色 T + 橄榄卡其短裤、厚下唇、小麦肤、北方/山东脸宽、约38岁成熟男 | 姿势、指向、站位、占画比例（表情仅稳定域内） |
| 禁止跨图漂移 | 一张粗黑框一张细框；一张宽脸一张韩范小脸；一张短裤一张运动裤；一张背身一张正脸；一张小石头有嘴/漩涡眼一张白圆眼；一张胶囊细长一张矮胖；**并排放不像同一人** | — |

**老杨多场景硬协议**：先 1 张预览门禁；同批每张复用同一段 Persona Feature Stability Lock + Outfit/Accessory Layer；表情只用 E0–E4 预设；手绘系每张传双全景；交付前做并排叠影快检（见 `persona-quick-checklist.md` X0–X6）；FAIL 只返修单张。

同图内多个小石头仍遵守上文「形体一致锁 + 眼睛/手臂严格锁」：**差异来自动作分工，不是换形体或换眼法**。

详细 persona 失败信号见 `physical-qa-checklist.md`、`handdrawn-qa-checklist.md` 与 profile 的 `persona-author-identity.md`。

## 多角色同图

同一主角色可以出现 2-4 个个体，但必须拉开至少 3 个维度：**整体缩放高度、朝向、倾斜、站位、手脚姿态、动作职责、所持道具**。禁止复制粘贴后平移位置。

### 形体一致锁（多人 CRITICAL）

多人 / 多 Agent 同图时，**每个个体的胶囊宽高比必须与 `primary-character-reference.png` 一致**（竖椭圆，约高:宽 = 1.8–2.2:1）：

- **只允许**：整体等比缩放（建议高度差 ±10–15%）、倾斜、朝向、站位、动作分工不同。
- **禁止**：A 细长竖条、B 矮胖圆球、C 横扁椭圆等**改变宽高比**的形体变异；禁止把「拉开差异」理解成换物种或换胖瘦。
- QA 快检：三个体并排时，把胶囊轮廓叠影对齐，**外轮廓形状应重合**，只有大小与角度不同。

### 眼睛严格锁

对照 `primary-character-reference.png`，每只主角色必须满足：

- **数量与形态**：恰好两只 **平面白圆眼**；可选 **tiny 黑点瞳孔** 居中；禁止 Logo 护目镜眼、anime 大眼、3D 眼窝、反光球体。
- **位置**：位于胶囊**上半部约 1/4 高度带**，两眼水平排列、左右对称。
- **尺寸**：单眼直径约为胶囊**宽度 12–18%**；两眼间距约为眼径 **1.2–1.8 倍**。
- **多人同图**：眼睛画法、瞳孔有无、大小比例**全批一致**，不得 A 有瞳孔 B 无瞳孔、或眼距 wildly 不同。

### 手臂与腿严格锁（与 Limbs Lock 一致）

- **数量**：每侧**恰好一条**臂 + 每侧**恰好一条**腿（或底缘正中双足布局）；牵引线 / 绳子 / 箭头不得与手臂粘连成第三只手。
- **臂锚点**：体侧 **上 1/3 高度带** 连续伸出；**禁止**断线悬浮、从头顶/眼睛下/底缘乱长。
- **腿锚点**：底缘左右或正中连续向下；**禁止**从体中部横出、悬浮断腿。
- **线宽与末端**：与 reference 同级细黑线，批内一致；末端仅小圆 / C 形 / 单线钩；**禁止粗臂、3D 圆管、分叉指、额外小手**。
- **无嘴**：小石头不靠嘴表达情绪；有嘴线即 FAIL。
- **动作**：臂必须参与递 / 接 / 贴 / 拉等叙事；腿参与站 / 跑 / 撑 / 踢等；禁止纯装饰站桩（装饰性测试仍适用）。

## 必须承担动作

主角色必须参与画面叙事。删掉主角色后隐喻仍成立，或主角色只是站旁边装饰，均视为失败。
