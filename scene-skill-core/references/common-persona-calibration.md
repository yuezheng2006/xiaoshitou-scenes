# 人像参考图校准协议

只要画面涉及**人像参考图、作者 persona、老杨、真人身份或需要保持同一人的角色**，本协议就是生成前的硬门禁。

## 适用范围

- 用户当前会话上传的人像参考图
- profile 内的 persona 全景、face-lock、flat IP 金样、spec、body 和动作参考图
- 老杨三视图、角色设定表、面部表情表、装备设定图
- 手绘、实物、知识卡、PPT、长卷中的 persona 出镜
- 单张生成、局部编辑和 2 张以上的批量生成

## 参考优先级

同一任务存在多份参考图时，按以下优先级处理：

1. 用户当前会话明确指定的最新人像参考图：身份、年龄、脸型和五官最高优先级。
2. 当前 profile 的模式主锚点：实体全景或手绘全景，决定渲染语言和全身比例。
3. 用户确认的 flat IP 金样对照组：`author-persona-flat-ip-sheet.png` + `author-persona-flat-ip-sheet-navy.png`，共同锁定最接近的脸、颧骨、年龄档、眼镜、发型、嘴唇和身体比例；藏蓝色只视为换色变体。
4. `face-lock`：多角度面相一致性和跨图稳定。
5. **批内金样**（同风格多场景、预览已验收）：`validated-batch-anchor-handdrawn.png`；锁定本批 likeness，不替代主锚。
6. `handdrawn-body` / `actions`：手绘比例、复杂姿态和小比例动作。
7. `spec`：面相偏移区和返修补强，不替代模式主锚点；若图片内部标注与主全景冲突，不采用其年龄、体重或服装文字。

如果不同参考图在年龄、脸型、服装或渲染语言上冲突，必须以用户当前明确要求为准，并在校准卡中记录覆盖关系；不得静默混用。

## 生成前校准卡（必须完成）

在 prompt 或内部生成计划中写出：

```text
人像参考图校准卡：
- 参考图：{文件名 / 当前会话参考}
- 身份锚点：{脸型、颧骨高度与面中结构、眼镜、发型、肤色、嘴唇、两侧颊胡茬分布、关键五官}
- 年龄锚点：{固定目标年龄与成熟度；同批同一视觉年龄档；不得用深皱纹代替成熟}
- 身材锚点：{身高观感、肩宽、腿身比例、体型}
- 服装配件：{服装、眼镜、鞋、发型、其他固定配件}
- 模式渲染：{实体 3D / 手绘黑线 / 其他 profile 规定}
- 本次允许变化：{姿势、表情、动作、站位、道具}
- 本次禁止变化：{换脸、换年龄、换发型、换眼镜、换装、泛化人脸}
- 校准结论：PASS / FAIL（FAIL 不得生成）
```

校准至少要回答：

- 生成的人是否仍是同一个人？
- 年龄是否符合目标年龄，而不是被皱纹、眼袋或疲态推老？
- 脸型、颧骨高度与面中结构、眼镜、发型、嘴唇、肤色和两侧颊胡茬分布是否稳定？
- 身材比例和服装是否来自正确模式参考？
- 人像的渲染语言是否符合当前模式？

## 生成中约束

- 人像参考图只用于身份和比例校准，不把参考图本身复制进画面。
- 不得把用户上传的人像参考图保存进公开仓库。
- 多张同批任务先生成 1 张预览；预览未通过校准，不得批量生成。
- 局部编辑只修改用户指出的偏差，保留已通过校准的脸型、眼镜、发型、服装和构图。

## 生成后校准检查

候选图必须与参考图并排检查：

1. 身份：脸型、五官位置、眼镜和发型仍能确认是同一人。
2. 年龄：符合目标年龄且与所有参考资产处于同一视觉年龄档；没有显嫩或显老漂移，也没有深皱纹、重眼袋、法令纹或疲态导致的年龄漂移。
3. 配件：眼镜、发型、服装和鞋子没有丢失或换款。
4. 比例：全身、大半身和小比例远景都保持正确身材观感。
5. 表情：表情变化没有改变脸型、年龄或人物身份。
6. 渲染：人像符合当前模式，不能变成 generic 人脸、照片级商业肖像、错误的 3D/手绘风格。

任一项 FAIL：标记 `CALIBRATION_REJECTED`，不得交付；只返修 FAIL 项，合格部分保持不变。

## 38 岁老杨的特别校准

老杨目标年龄固定为 38 岁：成熟、稳重、亲和，但不过老。所有实体全景、手绘全景、face-lock、表情图和动作图必须处于同一视觉年龄档；任何一张明显显嫩或偏大都视为年龄锁失败。成熟感来自骨相、眼神、体态和轻微胡茬；禁止用深额头纹、深法令纹、重眼袋、疲惫阴影或显老皱纹制造成熟感。
