# PPT 演讲模式

## 一句话

必须像 PPT 页面，但不能像廉价模板、培训讲义、信息图堆叠或商业海报。用于有当前 IP profile（可选 persona）前提的直播分享、课程课件、主题演讲和案例复盘，产出整套连续的手绘风 PPT 页面图。

## 什么时候用

用户明确要做 **PPT / 课件 / 直播分享页 / 主题演讲 / 整套演讲页面 / 按大纲逐页出图** 时进入本模式。

不适用于：

- 没有当前 IP profile 前提的普通商业汇报、数据报表、客户报价。
- 非手绘风格的发布会、科技风、商业模板风或杂志风 PPT。

## 工作流

```text
主题 / 大纲 / 逐字稿 / 旧 PPT
-> 理解演讲目标、受众和口播主线
-> 输出整套导演规划卡
-> 用户调整页数、节奏、密度、人物出现频率
-> 选 1-2 页代表页做样张
-> 确认字体、人物和信息密度
-> 分批生成页面
-> 单页 QA 和整套 QA
-> 返修问题页
```

硬规则：

- 不先批量出图，必须先出导演规划卡。
- 不在没有规划的情况下逐页自由发挥。
- 不把每一页都做成知识卡或高密度机制图。

## 导演规划卡（出图前必须先给）

```text
页码：
页面标题 / 暂定标题：
页面类型：
视觉权重：anchor / dense / breathing
沟通任务：
核心观点：
主视觉：
图文比例：
是否放主角色 / persona：
人物角色/动作职责：
图内文字策略：
前后页衔接：
QA 风险：
```

用户确认导演规划后，每页生成前形成 page card：

```text
页码：
页面类型：
视觉权重：
沟通任务：
核心观点：
图内标题：
图内文字：
主视觉：
主角色 / persona 职责：
禁止项：
返修重点：
```

page card 是生成提示词和出图后 QA 的依据。

## 页面类型库

| 页面类型 | 用途 |
| --- | --- |
| 封面页 | 定主题、定气质、建立第一印象 |
| 目录/路线页 | 告诉听众接下来怎么走 |
| 章节转场页 | 切换部分，帮助听众换脑子 |
| 场景页 | 用画面讲真实情境 |
| 痛点页 | 放大一个问题，引发共鸣 |
| 大判断页 | 一句话立观点 |
| 类比页 | 用隐喻解释复杂概念 |
| 机制页 | 解释系统如何运转 |
| 案例页 | 讲具体案例的背景、动作、结果 |
| 时间线/路线图页 | 展示阶段、演进、计划 |
| 小结页 | 阶段收束，让听众记住 |
| 金句页 | 强化一句核心记忆点 |
| 结束页 | 回扣主题、给行动建议、感谢 |

页面类型由内容决定，不由固定模板决定；不必用完所有类型。

## 视觉权重和节奏

- `anchor`：封面、章节、关键判断、重要转折、结尾。
- `dense`：机制、数据、流程、证据。
- `breathing`：过渡、故事、金句、小结、低密度场景。

节奏规则：

- 不连续 3 页同一种页面类型。
- 不连续 3 页 `dense`。
- 每页只承担一个沟通任务。
- 机制页可以密，但必须被场景页、转场页或大判断页隔开。
- 图比字多，文字服务口播，不替代口播。
- 每 8-12 页至少有一次小结或明显转场。
- 结束页必须回扣开头。

## 视觉规范

- 默认风格：稳定当前 IP profile + 白底手绘图解 + PPT 页面纪律 + 整页图。
- 16:9，读取 `references/common-modes-and-sizes.md`。
- 主标题稳定、粗黑、清晰；正文/小标签尽量统一；手写字只作少量批注，不承担主信息；中文字太多就拆页，不硬塞。
- 默认白底或近白底，黑灰线稿为主；橙色只做重点/路径/批注；蓝色只做结构线/连接线；不使用大面积渐变或商业模板色。

## 主角色 / Persona 的使用

- 主角色是核心出镜角色，必须基于当前 profile 的角色锚点和维度锁，禁止漂移。
- 人物出现时必须有动作职责（讲解、指向、演示、收束），不是站旁边装饰；先做 `common-qa-repair.md` 的装饰性测试。
- 不要每页都放大人物；封面/身份页/关键转折页/结束页可以大，机制页/场景页角落小，数据页一般不出现。
- 多个主角色协作时遵守 `common-character-lock.md` 和当前 profile 的差异锁。
- Persona 出镜按 `common-persona-routing.md` 和当前 profile 的 persona 文件处理。触发后，persona 可以在封面、身份/信任页、关键判断页、讲解页、小结页和结束页承担讲者、主讲 IP、批注者或收束者职责。
- PPT 中的 persona 渲染风格由当前 profile 的 persona 文件决定，默认随手绘视觉语言走扁平黑线；是否出现在每页由导演规划控制，不能连续多页放大 persona，也不能让 persona 替代页面沟通任务。
- 未触发 persona 时，不把真人或作者形象加入页面。

## QA

单页 QA：

- 是否符合 page card。
- 是否只有一个核心任务。
- 主标题字体是否稳定、清晰。
- 图内文字是否过多、过小、错字。
- 主角色 / persona 是否像各自锚点、是否有职责。
- 画面是否像 PPT 页面，而不是知识卡、信息图海报或培训讲义。
- 橙色是否过量。

整套 QA：

- 是否像同一个设计系统，字体系统是否乱。
- Persona 年龄、气质、服装、发型是否漂移，主角色形体是否漂移。
- 是否连续多页高密度、页面类型是否有变化。
- 人物是否出现过多或过大。
- 是否有足够转场、场景和 breathing 页。
- 结尾是否回扣开头。

不合格先回到对应页的 page card 重新对齐，再局部返修或重生成；不要跳过导演规划直接批量重做。返修输出统一用 `common-qa-repair.md` 的格式。
