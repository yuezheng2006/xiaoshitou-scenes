# 手绘图构图模式与原创规则

## 基础结构类型

选择一种结构即可，不要混太多。

### Workflow 流程

适合：输入 -> 处理 -> 输出、内容生产、Agent 工作流、自动化链路。

画法：左侧输入，中间是主角色参与的怪机器或处理动作，右侧输出；橙色箭头表达主流向。

### 系统局部

适合：信息来源、过滤器、数据库、渲染器、Agent 系统局部。

画法：只画 3-5 个核心模块，主角色参与其中一个关键动作。

### 前后对比

适合：混乱/有序、手动/自动、分散/收拢、焦虑/稳定。

画法：左侧混乱，右侧稳定，中间用橙色箭头连接；主角色可以在中间承担转换动作。

### 角色状态

适合：用户痛点、创作者状态、工具太多、信息焦虑、从卡住到跑起来。

画法：2-4 个小状态，每个状态一个短标注；不要做成密集漫画。

### 概念隐喻

适合：内容工厂、信息仓库、脑内黑盒、工作流机器、自动日报。

画法：一个大的怪物件或机器，少量输入，一个输出；主角色在里面执行关键动作。

### 方法分层

适合：方法论框架、系统层级、能力栈、内容系统分层。

画法：一层层盒子或积木，不要正式金字塔；主角色在旁边搬砖、搭建或调整。

### 地图路线

适合：从想法到上线、用户路径、内容承接路径、学习路线。

画法：一条弯曲路径，少量节点，主角色牵线或沿路走。

### 小漫画分镜

适合：失败到成功、真实过程、使用前后变化、踩坑复盘。

画法：2-4 个小场景，每格只表达一个动作；保持留白。

### 双 IP 白板协作（persona 触发时优先）

适合：Agent 工作流、方法论、系统结构；默认 profile 触发老杨时使用。

画法：老杨画结构 / 写批注 / 指向模块（约 40–55%）；小石头拉线 / 搬模块 / 守门 / 塞入流转槽（执行 Agent）。读 `persona-scene-patterns.md` 选「白板拆解」场景。

## 原创隐喻生成法

每次都从当前内容重新发明隐喻，不能照搬旧图。

1. 把抽象概念换成一个物理动作：卡住、漏掉、变重、分层、沉淀、折叠、拆包、回流、对齐、拦截。
2. 把系统结构换成一个现代结构物件：任务队列盒、权限闸口、协作看板、自动化流转槽、状态仪表、标签栈、连接线阵列、模块箱、同步桥。
3. 让主角色承担动作：卡在装置里、拉错线、守闸口、搬运、修补、分层、记录、把东西塞进流转槽。

## 可用物件池

- 任务队列盒、权限闸口、协作看板、自动化流转槽、状态仪表、标签栈、连接线阵列、模块箱、同步桥、消息收纳盒、智能筛选板、圆角设备卡、轻量工具台。
- 用时只选 1-2 个，不要堆满。

## 主角色动作池

- 拉、扛、塞、捞、压、分层、对齐、剪、拧、守、推、接、拆、标记、回收。
- 动作要服务核心意思，不要为了怪而怪。

## 反复刻规则

手绘图可以参考上游 Ian / helloianneo 手绘解释工作流（1.0）的风格密度、留白和怪诞感，但不要复刻旧案例构图。

不要默认复用这些旧构图：

- 传送带断点。
- 角色在内容机器里拉判断杆。
- 任务队列盒分层流量 / 信任 / 转化。
- 切素材鱼。
- 承接路径牵线。
- 三层信息源拉线。
- 话术工具台贴状态标签。
- 常见坑路径举牌。

同类主题也要换新隐喻。例如「承接路径」不一定画路线，可以画主角色把内容尾巴接到门把手；「一鱼多吃」不一定画鱼，可以画主角色把一个纸团压成几种形状。
