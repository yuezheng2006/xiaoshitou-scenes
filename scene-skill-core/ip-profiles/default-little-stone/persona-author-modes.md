# 老杨 · 各模式职责与双 IP 协作

互动场景类型见 `references/persona-scene-patterns.md`。本文只定义**各模式下老杨与小石头谁做什么、占多少画面**。

## 双 IP 总原则

```text
老杨讲、小石头干 — 每张图必须有双向动作关系
```

| 模式 | 老杨 | 小石头 | 画面重心 |
| --- | --- | --- | --- |
| 知识卡 / PPT / 手绘 | 主讲 40–55% | 1–3 执行 Agent 25–40% | 结构 + 互动 |
| 实物图 / 长卷 | 调度 25–40% | 物理动作 30–45% | 物件小现场 |
| 观点 / 9:16 海报 | 主讲占主视觉 | 0–1 执行点缀 | 一句话传播 |

## 实物图模式

- **重心**：真实物件 > 小石头物理动作 > 老杨调度。
- **老杨**：递工具、看反馈、扶物件、写旁注、指向节点。
- **小石头**：拉、挡、推、接、整理等核心物理动作。
- **渲染**：老杨轻 Q 3D；小石头 flat 2D；同一白色摄影棚空间。
- **例外**：用户要求「老杨大判断」类实物隐喻时，老杨可升至 40–50%，仍须小石头执行点缀。

推荐结构：

```text
真实物件主现场 + 小石头执行动作 + 老杨递工具/看反馈/做旁注
```

## 手绘图模式

- **重心**：结构解释 + 双 IP 协作。
- **老杨**：画结构、写批注、指向模块（40–55%）。
- **小石头**：拉线、搬模块、守门、塞入流转槽。
- **渲染**：扁平黑线；读 spec + handdrawn；不用 3D 描述。

推荐结构：

```text
老杨画结构/写批注 + 小石头拉线/搬模块/守门
```

## 知识卡模式

- **重心**：标题 + 模块 + 阅读顺序。
- **老杨**：贴近标题或结论，主讲 / 指向 / 总结。
- **小石头**：1–3 个在步骤 / 案例 / 风险区搬卡、贴标签、标风险。
- **默认**：方法论 / 步骤 / 流程 / 对比类 → 老杨 + 2–3 小石头；不能只有老杨和文字。
- **渲染**：同手绘（spec + handdrawn）。

推荐结构：

```text
老杨主讲/指向 + 小石头在模块里搬卡/标风险/递交结果
```

## PPT 演讲模式

- **重心**：单页一个沟通任务；像 PPT 页面，不像知识卡海报。
- **老杨**：封面 / 大判断 / 方法 / 收束页可占主视觉；讲解、批注、收束。
- **小石头**：执行点缀或模块内动作；不能连续多页只有老杨大头像。
- **渲染**：同手绘（spec + handdrawn）。
- 导演规划与 page card 规则见 `references/ppt-presentation-mode.md`。

## 彩蛋长卷

- 每节点：真实物件 + 小石头动作 + 老杨递节点卡 / 旁注。
- 老杨可小比例出现，须满足 [persona-author-identity.md](persona-author-identity.md) 小比例识别线。

## 常见互动动作（任选，须双向）

- 老杨递便签/卡片/工具 → 小石头接住执行
- 小石头递交结果 → 老杨检查/批注
- 老杨扶物件/画结构 → 小石头拉/挡/推/搬模块
- 老杨指向标题/结论 → 小石头在步骤区搬卡/标风险

## 禁止（模式层）

- 触发后只有老杨、没有小石头；或只有小石头、老杨无动作
- 头像 / 专访封面 / 个人品牌海报（任何模式）
- 老杨替代小石头承担全部物理执行
- 未触发时自动加入老杨
