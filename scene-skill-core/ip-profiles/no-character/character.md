# Character Profile: none

## 核心定位

`$IP=none`：**无品牌固定角色**，不是「画面禁止出现人」。

默认以物件、白板结构、短标签叙事。只有当次信息分工需要「谁焦虑 / 谁操作 / 谁介入」时，才可加 1–2 个功能性简笔人。

## 资产

- 无设定图、无校准图、无动作扩展。
- 质量不稳时，可选用当前模式的通用母版 / 示例作**风格**参考，但不得当作小石头或老杨身份锚点。

## 填入 `{IP_DESC}`

```text
No branded IP character (NOT Little Stone — no #f39800 capsule body, white circular eyes as brand mascot, or primary-character-reference look; NOT Lao Yang / author persona).
Object- and structure-focused composition is default.

When the shot needs an information role for a user or AI, you MAY add 1-2 simple generic figures:
- generic stick-figure user (worry, action, at desk) — small, sketchy, no detailed portrait
- simple robot labeled AI — round, friendly, not a branded mascot
Figures are optional tools for information roles, not required in every image.
If figures are not needed, use objects, arrows, and short Chinese labels only.
```

## 填入 `{IP_STYLE_ADAPT}`

留空。

## 功能性简笔角色（按需）

| 何时用 | 画什么 | 禁什么 |
| --- | --- | --- |
| 信息分工需要「谁焦虑/谁操作」 | 简笔用户、简笔机器人（标 AI） | 小石头外貌、老杨肖像、品牌吉祥物 |
| 情绪 / 用户 vs 工具对比 | 同上，小比例，物件为主 | 人物占画面 >30% |
| 纯步骤 / 物件序列 | 通常不画人 | 为凑热闹加人 |

## 失败信号

- 偷偷画回 `#f39800` 胶囊小石头或老杨脸
- 把 none 理解成「禁止一切人形」，导致本该有「操作者」的信息分工缺失
- 精细五官肖像、品牌 mascot、代言海报感

## Character Lock

none 模式不声明品牌 Character Lock。若同批多张，保持简笔演员画法一致即可。
