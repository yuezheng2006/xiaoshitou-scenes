```

  - 小石头：L1 计数 + E1 眼睛 + E2 无嘴
  - 老杨：P1 眼镜 + P3 脸型 + P6 可见性
  - 跨图：X1 形体一致 + X4 核心特征一致

  - 小石头：L2 锚点 + L3 线质 + L4 末端
```

## 🎁 关键洞察

### 1. 品牌化叙事的合理性

**之前的假设**：公开开源项目 → 需要去品牌化  
**实际情况**：内部开源 → 品牌化是资产

**重新理解**：
- 企业 = 全场景 AI 音乐娱乐公司（不仅仅是 KTV）
- AI native 研发场景 ≠ KTV 门店服务场景
- 品牌化增强团队凝聚力和 IP 识别度

### 2. 复杂度来源分析

**过度精确的标准**：
- "38 岁"精确值 → 模型无法精确控制
- "上 1/3" 边界模糊 → Agent 主观判断
- 11 个资产冗余 → "何时用 spec/face-lock" 困惑

**解决方案**：
- 精确值改为范围（35-40 岁）
- 边界量化（上 1/4-2/5 = 25-40%）
- 资产精简（合并功能重叠项）

### 3. 质量与效率的平衡

**不是降低标准**：
- Tier 1 保持严格（核心识别不变）
- Tier 2 保证质量（整体质量不降）
- Tier 3 追求卓越（不强制）

**而是合理分级**：
- 只有真正影响 IP 识别的才是 CRITICAL
- 细节优化降级为 IMPORTANT / NICE-TO-HAVE
- 交付阻断率降低，但质量门禁保留
