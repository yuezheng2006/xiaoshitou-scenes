# 母版选择与拦截协议

## 母版 PNG 状态

标准母版文件名为 `assets/examples/01-alignment-pull-in.png` … `07-brand-evolution-long-scroll.png`。根目录 `examples/images/` 与 Skill 包 `thunderstone-xiaoren/assets/examples/` 保持同步。

1. 仍必须锁定母版**类型**（01–07）并填写锁定字段。
2. 生成前 Read 对应 PNG + 本文档抽象骨架 + `style-dna.md` + `assets/brand/` 参考图。
3. 若 PNG 缺失（Fork 未同步），以本文档各节**抽象骨架** + `style-dna.md` 比例/留白 + `assets/brand/` 小石头参考图为质量锚点。

下文「目标画面」描述的是母版类型的**目标规格**（供重生成 PNG 与无 PNG 时对齐），不是可机械复制的布局。

本文件用于修复两个关键风险：

- 元素清单化：模型把主题词、物件和标签当成清单堆进画面。
- 母版复刻化：模型把高质量案例当成可描摹模板，直接复制手机、纸条、挡板、沙漏等空间拓扑。

正确做法是先选择一个案例作为**质量锚点**，再抽取它的抽象不变量，最后为当前内容写出必须变异的地方。母版不是可复制画面，也不是空间模板；它只提供质量标尺和动作类型。

## 母版使用总原则

每次使用母版必须同时满足：

- **保留不变量**：比例克制、留白、真实物件质感、小石头参与核心动作、标签少而准；**同批配图还须保留 Character Lock（同一小石头）**。
- **必须变异**：主物件、空间方向、动作关系、道具组合、标签位置、视角或叙事重心中，至少 3 项必须和母版不同。
- **先读懂再好看**：不看说明，读者 3 秒内也能说出“小石头正在被什么困住 / 挡住 / 拉扯 / 审查 / 筛掉”。

如果生成图第一眼像某张案例的换皮版，就判失败，即使它背景、标签和小石头都对。

## 标准模式母版索引

### 01 alignment-pull-in

文件：

```text
assets/examples/01-alignment-pull-in.png
```

目标画面（母版类型规格，不是可复制布局）：

- 左侧小石头用力拉黑色挂绳 / 弹力绳。
- 右侧黑色风琴文件夹 / 档案夹被绳勒住，夹内露出白纸。
- 标签贴在文件夹正面（如「对齐一下」「再开一次」「谁拉的」）。
- 纯白底，配件极少，留白大。

适合：

- 会议、同步、对齐、临时拉人、需求被再次打开、下班前被叫回。

抽象骨架：

- 多个外部请求从主物件一侧施加拉力。
- 小石头被拉向任务入口，或正在把某物「再拉开 / 再对齐」。
- 留白很大，配件很少。

可复用关系：

- 「任务 / 会议 / 需求」作为被拉开的物件或入口。
- 小石头与挂绳、胶带、文件夹、门把手发生拉力。
- 标签在来源或主物件附近，不要每个物件都贴满字。

失败信号：

- 直接复制「左小石头拉绳 + 中/右文件夹 + 标签贴夹面」的拓扑。
- 主物件过大，变成商品特写。
- 线太多，像蜘蛛网。
- 纸条太多，像信息流合集。
- 误画成电脑 + 会议卡片旧版 generic 拓扑（本仓库母版已是文件夹方案）。

### 02 info-overload

文件：

```text
assets/examples/02-info-overload.png
```

目标画面（母版类型规格，不是可复制布局）：

- 左侧黑色热敏打印机持续吐出长条点歌 / 收据纸。
- 小石头用真实纸板挡在纸流前方。
- 可选 KTV 语境小吊牌（如右上角「KTV」），但不是 UI 截图。
- 标签点出过载话术（如「消息太多」「马上回」「谁点的」）。

适合：

- 消息过载、点歌条涌出、任务涌出、已读不等于完成、催促。

抽象骨架：

- 一个真实物件作为「输出口 / 涌出口 / 源头」。
- 小石头正在挡、推、接、压住或整理涌出的东西。
- 画面表达「东西出来太多，人挡不住」。

可复用关系：

- 主物件是「出口」，不是信息展示屏；出口可以是打印机、抽屉、信封，不必是手机。
- 小石头必须挡、推、接或整理，不是站着看。
- 标签只点出 2-4 个真实话术。

失败信号：

- 直接复制「左打印机 + 中间纸流 + 小石头挡纸板 + 右上 KTV 吊牌」的拓扑。
- 当前内容不是消息 / 点歌 / 涌出经验，却仍然强行用热敏打印机。
- 出现聊天 UI 或截图。
- 纸流文字太多、标签过密。
- 主物件过大，画面像设备广告。
- 误画成「左手机 + 中间纸条 + 挡板 + 沙漏」旧版 generic 拓扑。

### 03 problem-knot-alert

文件：

```text
assets/examples/03-problem-knot-alert.png
```

目标画面（母版类型规格，不是可复制布局）：

- 黑色粗线缆横穿画面，中央打成结。
- 小石头双手握住结附近，正在排查 / 解结。
- 标签可竖排在结旁（如「又断了」「小问题」「先排查」）。
- 纯白底，无 KTV 包厢背景。

适合：

- 报警、返工、打结、问题回流、边界情况、设备线缆故障、下班失败被拽回。

抽象骨架：

- 一个真实长条物或路径形成卡点、结、回流或缠绕。
- 小石头的动作集中在卡点上。
- 画面表达「本来应该顺着走，但在这里卡住 / 回来了」。

可复用关系：

- 线缆、软管、挂绳就是冲突的物理路径。
- 小石头动作集中在「结」上。
- 标签贴近问题点，不要把全流程画出来。

失败信号：

- 直接复制「横穿黑线缆 + 中央结 + 小石头双手握结 + 右侧竖排标签」的拓扑。
- 线缆变成装饰曲线，和小石头没有力学关系。
- 标签散太远，看起来像海报注释。
- 配件过多，冲淡「打结」的核心现场。
- 误画成桌游卡牌、解谜 MASTER 卡（品牌误认）。

### 04 review-rework

文件：

```text
assets/examples/04-review-rework.png
```

目标画面（母版类型规格，不是可复制布局）：

- 中央纸堆 / 文档，上面有红圈、红线下划线。
- 小石头持放大镜检查，另一手持红笔。
- 左侧橡皮章 / 改字章（如「再改改」），左侧可有气泡短标签。
- 表达「表面完成了，还要被复核 / 改稿」。

适合：

- 审查、校对、复核、返工、验收、谁来兜底。

抽象骨架：

- 一个真实检查工具把内容局部放大、压住或框住。
- 小石头在检查区域里执行修、查、改、盖、圈。
- 画面表达「表面完成了，但还要被复核 / 兜底」。

可复用关系：

- 放大镜、红笔、印章是「被审查 / 要再看」的物理工具。
- 小石头必须在被检查或检查别人之间形成清楚关系。
- 标签要像审查意见，不像流程名称。

失败信号：

- 直接复制「中央纸堆 + 小石头持放大镜 + 左侧改字章 + 左侧气泡标签」的拓扑。
- 放大镜太大，主体变成单一物件特写。
- 小石头只是站在镜子里，没在做动作。
- 纸堆太满，像办公素材拼贴。

### 05 automation-relabel

文件：

```text
assets/examples/05-automation-relabel.png
```

目标画面（母版类型规格，不是可复制布局）：

- 大尺寸挂绳工牌，牌面有多色标签块（如「自动化」「AI 先做」「人来兜底」「谁改的」）。
- 小石头一手扶工牌，一手拿修正带改字 / 贴标签。

- 工牌牌面若印 Logo，**必须** 原样使用 `assets/brand/thunderstone-logo-core.png` 几何，禁止自画。
- 横向铺开，配件极少。

适合：

- AI 先做、人来兜底、岗位重组、自动化牵引、身份被重新贴标签。

抽象骨架：

- 一个真实标签、身份牌或纸带正在重新定义某个东西。
- 小石头用身体阻止、拉回、撕开或按住这个标签变化。
- 画面表达「系统给了新名字，但人还在承担后果」。

可复用关系：

- 标签 / 纸带 / 工牌代表系统给人的新定义。
- 小石头和挂绳 / 标签发生拉扯或改字。
- 主体横向铺开，配件极少。

失败信号：

- 直接复制「左小石头 + 中挂绳工牌 + 修正带改字 + 多色牌面标签」的拓扑。
- 做成 AI 工具广告、机器人图或胸口 Logo 胸标。
- 写太多「自动化 / 岗位 / 重组」概念词。
- 小石头站在旁边，无法看出它在抵抗或兜底。
- 在非办公 / 权限场景误加工牌。

## 必填母版锁定字段

每张标准模式图在生成前必须写出：

```text
母版类型（01–06 或 07）：
Character Lock（2 张及以上）：
抽取的不变量：
当前内容的变异点：
画面 3 秒读懂句：
当前内容适配：
要避免的失败信号：
```

如果这四项说不清，不生成，先重写 shot list。

变异点至少写 3 个，例如：

- 换掉主物件类别。
- 改变左右/中心空间关系。
- 改变小石头动作，不再做同一种推/拉/挡。
- 删除母版里的标志性配件。
- 改变标签出现方式。
- 改变视角或物件姿态。

## 出图拦截规则

生成后的第一张图永远只是候选图，不是交付图。必须和所选母版做视觉比对：

- 主体比例是否接近母版，而不是商品特写或缩略摆件。
- 留白是否接近母版，而不是元素铺满。
- 真实物件是否像同一张摄影棚照片。
- 小石头是否承担母版级的物理动作。
- 标签是否像情绪按钮，而不是概念说明。
- 是否出现“主题元素清单化”：把电脑、手机、纸条、线、标题都堆上去。
- 是否出现“母版复刻化”：主体、方向、配件、小石头动作和标签位置都像旧案例换字。
- 是否 3 秒读懂：不看解释，读者是否能说出画面冲突。

只要出现“元素清单化”“母版复刻化”“不像母版正文配图”或“看不懂冲突”，就必须重写提示词并重生成，不得交付为最终图。
